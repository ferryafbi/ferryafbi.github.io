Rangkuman Materi CCNA-1: Address Resolution

Ferry Ananda Febian

Jan 4, 2023 • 3 min read

Address Resolution adalah proses mengubah alamat logis (seperti alamat IP) menjadi alamat fisik (seperti alamat MAC) yang digunakan oleh perangkat jaringan untuk mengirimkan paket data ke perangkat tujuan. Proses ini diperlukan karena setiap perangkat jaringan hanya dapat menerima dan mengirimkan paket data melalui interface fisiknya (misalnya, kartu jaringan), yang memiliki alamat MAC unik.

Untuk menentukan alamat MAC dari perangkat tujuan, perangkat yang akan mengirimkan paket data terlebih dahulu mengirimkan request ke jaringan untuk mencari tahu alamat MAC dari perangkat tujuan. Apabila perangkat tujuan ditemukan, maka perangkat tersebut akan mengirimkan balasan yang berisi alamat MAC-nya ke perangkat yang meminta. Setelah perangkat pengirim menerima alamat MAC perangkat tujuan, maka paket data dapat dikirimkan ke perangkat tujuan melalui interface fisiknya.

Proses address resolution terdiri dari beberapa tahap, yaitu:

  • Perangkat pengirim mengirimkan request ke jaringan untuk mencari tahu alamat MAC perangkat tujuan.
  • Perangkat tujuan menerima request dan mengirimkan balasan yang berisi alamat MAC-nya ke perangkat pengirim.
  • Perangkat pengirim menerima alamat MAC perangkat tujuan dan mengirimkan paket data ke perangkat tujuan melalui interface fisiknya.

Dengan menggunakan proses address resolution, perangkat jaringan dapat saling berkomunikasi dengan efisien, meskipun hanya memiliki alamat logis (seperti alamat IP) yang tidak dapat digunakan langsung untuk mengirimkan paket data.

MAC and IP

Alamat MAC (Media Access Control) adalah alamat fisik yang terdapat pada setiap perangkat jaringan, sedangkan alamat IP (Internet Protocol) adalah alamat logis yang digunakan untuk mengidentifikasi perangkat jaringan di dalam jaringan. 

Berikut ini adalah perbedaan utama antara alamat MAC dan alamat IP:

  • Fungsi: Alamat MAC digunakan untuk mengirimkan paket data antar perangkat jaringan pada jaringan lokal (local area network atau LAN), sedangkan alamat IP digunakan untuk mengirimkan paket data antar jaringan (internet).
  • Penggunaan: Alamat MAC hanya digunakan di dalam jaringan LAN, sedangkan alamat IP dapat digunakan di jaringan manapun yang terhubung ke internet.
  • Format: Alamat MAC terdiri dari enam byte atau 48 bit yang ditulis dalam bentuk hexadecimal (misalnya, 00:11:22:33:44:55), sedangkan alamat IP terdiri dari empat blok angka yang dipisahkan oleh titik (misalnya, 192.168.0.1).
  • Unik: Setiap perangkat jaringan memiliki alamat MAC yang unik dan tidak dapat digunakan oleh perangkat lain, sedangkan alamat IP dapat digunakan oleh beberapa perangkat jaringan dengan menggunakan teknik NAT (Network Address Translation).
  • Durasi: Alamat MAC tidak dapat diubah setelah perangkat jaringan dibuat, sedangkan alamat IP dapat diubah sesuai kebutuhan.

Kesimpulannya, alamat MAC digunakan untuk mengirimkan paket data antar perangkat jaringan pada jaringan lokal, sedangkan alamat IP digunakan untuk mengirimkan paket data antar jaringan dan mengidentifikasi perangkat jaringan di dalam jaringan.

ARP

ARP (Address Resolution Protocol) adalah protokol yang digunakan untuk mengubah alamat logis (seperti alamat IP) menjadi alamat fisik (seperti alamat MAC) yang digunakan oleh perangkat jaringan untuk mengirimkan paket data ke perangkat tujuan. Protokol ini digunakan untuk menyelesaikan proses address resolution di dalam jaringan lokal (local area network atau LAN).

Tujuan utama dari ARP adalah untuk memungkinkan perangkat jaringan saling berkomunikasi dengan efisien, meskipun hanya memiliki alamat logis (seperti alamat IP) yang tidak dapat digunakan langsung untuk mengirimkan paket data. Dengan menggunakan ARP, perangkat jaringan dapat dengan mudah mencari tahu alamat MAC perangkat tujuan yang dibutuhkan untuk mengirimkan paket data ke perangkat tersebut.

Secara garis besar, proses yang dilakukan oleh ARP terdiri dari beberapa tahap, yaitu:

  • Perangkat pengirim mengirimkan request ke jaringan untuk mencari tahu alamat MAC perangkat tujuan.
  • Perangkat tujuan menerima request dan mengirimkan balasan yang berisi alamat MAC-nya ke perangkat pengirim.
  • Perangkat pengirim menerima alamat MAC perangkat tujuan dan mengirimkan paket data ke perangkat tujuan melalui interface fisiknya.

Dengan demikian, ARP memiliki fungsi penting dalam proses komunikasi di dalam jaringan lokal, yaitu dengan memungkinkan perangkat jaringan saling berkomunikasi dengan menggunakan alamat logis (seperti alamat IP) yang lebih mudah dikenali oleh manusia.

Neighbor Discovery

IPv6 Neighbor Discovery adalah proses yang digunakan oleh perangkat jaringan untuk menemukan perangkat lain yang terhubung dalam jaringan yang sama, serta menentukan alamat dan informasi lain yang dibutuhkan untuk berkomunikasi dengan perangkat tersebut. Proses ini merupakan bagian dari protokol IPv6 yang digunakan untuk mengelola komunikasi di dalam jaringan.

Secara garis besar, proses IPv6 Neighbor Discovery terdiri dari beberapa tahap, yaitu:

  • Perangkat jaringan mengirimkan request multicast ke jaringan untuk mencari tahu apakah ada perangkat lain yang terhubung dalam jaringan yang sama.
  • Perangkat lain yang terhubung dalam jaringan yang sama akan mengirimkan balasan yang berisi informasi tentang diri mereka (seperti alamat IP dan MAC).
  • Perangkat jaringan yang mengirimkan request akan menerima balasan dari perangkat lain dan menyimpannya di dalam tabel neighbor untuk digunakan sebagai referensi ketika akan berkomunikasi dengan perangkat tersebut di kemudian hari.

Dengan menggunakan proses IPv6 Neighbor Discovery, perangkat jaringan dapat dengan mudah menemukan dan berkomunikasi dengan perangkat lain yang terhubung dalam jaringan yang sama, sehingga proses transmisi data dapat berlangsung dengan efisien.
