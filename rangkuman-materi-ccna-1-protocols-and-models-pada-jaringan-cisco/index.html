Rangkuman Materi CCNA-1: Protocols and Models Pada Jaringan Cisco

Ferry Ananda Febian

Jan 4, 2023 • 6 min read

Protocols and Models

Protocols dan Models adalah konsep yang sangat penting dalam dunia jaringan komputer, terutama jika Anda bekerja dengan perangkat Cisco. Protocols merujuk pada aturan atau standar yang ditetapkan untuk berkomunikasi antar perangkat dalam suatu jaringan. Misalnya, protokol TCP/IP adalah protokol yang sering digunakan untuk berkomunikasi antar perangkat dalam jaringan internet.

Models yang dimaksud adalah kalimat yang merujuk pada cara sebuah jaringan dibangun atau diatur. Misalnya, model OSI (Open Systems Interconnection) adalah model standar yang menjelaskan bagaimana data dapat dikirimkan antar perangkat jaringan melalui lapisan-lapisan yang berbeda. Model Cisco yang terkenal salah staunya adalah model Hierarchical Network Design, yang menggunakan pendekatan terstruktur untuk merancang jaringan komputer.

Protocols dan models sangat penting dalam dunia jaringan komputer karena mereka memungkinkan perangkat yang berbeda untuk berkomunikasi dan bekerja sama dengan baik. Tanpa standar yang jelas, jaringan akan menjadi tidak terorganisir dan sulit untuk diatur.

The Rules

Berikut adalah beberapa aturan yang harus diikuti dalam protokol dan model jaringan Cisco:

  • Menggunakan model OSI (Open Systems Interconnection) sebagai acuan untuk mengimplementasikan protokol jaringan. Model OSI membagi jaringan komputer menjadi 7 lapisan yang saling terkait, dan menentukan bagaimana setiap lapisan harus berkomunikasi dengan lapisan lainnya.
  • Menggunakan protokol TCP/IP (Transmission Control Protocol/Internet Protocol) sebagai protokol standar untuk pertukaran data di jaringan. TCP/IP merupakan protokol yang paling umum digunakan di Internet dan jaringan lokal.
  • Menggunakan perangkat keras jaringan yang sesuai dengan standar industri, seperti switch, router, dan bridge. Perangkat ini digunakan untuk menghubungkan dan mengatur lalu lintas jaringan.
  • Menggunakan protokol routing yang sesuai untuk menentukan jalur terbaik bagi paket data untuk sampai ke tujuannya. Protokol routing seperti OSPF (Open Shortest Path First) dan BGP (Border Gateway Protocol) digunakan untuk menentukan jalur terbaik bagi paket data yang akan dikirimkan di jaringan.
  • Menggunakan protokol keamanan jaringan seperti SSH (Secure Shell) dan SSL (Secure Sockets Layer) untuk menjaga keamanan data yang dikirimkan di jaringan. Protokol ini digunakan untuk mengenkripsi data sehingga hanya orang yang memiliki kunci enkripsi yang dapat membaca datanya.

Protocols

Protokol adalah aturan yang mengatur cara berkomunikasi antar perangkat dalam suatu jaringan. Dalam jaringan Cisco, terdapat banyak protokol yang digunakan, di antaranya adalah:

  • Internet Protocol (IP): protokol ini digunakan untuk mengirimkan paket data dari satu perangkat ke perangkat lain dalam jaringan.
  • Transmission Control Protocol (TCP): protokol ini menjamin bahwa paket data yang dikirimkan diterima oleh perangkat tujuan dengan benar. Jika terjadi kesalahan saat pengiriman, TCP akan mengirimkan pesan kesalahan kembali ke perangkat pengirim agar paket data tersebut dikirim ulang.
  • User Datagram Protocol (UDP): protokol ini juga digunakan untuk mengirimkan paket data dari satu perangkat ke perangkat lain dalam jaringan. Namun, berbeda dengan TCP, UDP tidak menjamin bahwa paket data yang dikirimkan akan diterima dengan benar oleh perangkat tujuan.
  • Dynamic Host Configuration Protocol (DHCP): protokol ini digunakan untuk memberikan alamat IP dinamis kepada perangkat-perangkat dalam jaringan.
  • Domain Name System (DNS): protokol ini digunakan untuk mengkonversi nama domain ke alamat IP.
  • Simple Network Management Protocol (SNMP): protokol ini digunakan untuk mengelola dan memonitor perangkat-perangkat dalam jaringan.
  • Network Time Protocol (NTP): protokol ini digunakan untuk menyinkronkan waktu pada perangkat-perangkat dalam jaringan.
  • Secure Shell (SSH): protokol ini digunakan untuk menjalankan perintah pada perangkat jaringan secara aman melalui jaringan.
  • File Transfer Protocol (FTP): protokol ini digunakan untuk mengirim dan menerima file melalui jaringan.
  • Hypertext Transfer Protocol (HTTP): protokol ini digunakan untuk mengirimkan halaman web dari server ke browser.

Protocol Suites

Protokol suite adalah sekumpulan protokol yang digunakan untuk berkomunikasi di dalam jaringan komputer. Pada jaringan Cisco, protokol suite yang paling umum digunakan adalah protokol TCP/IP (Transmission Control Protocol/Internet Protocol). TCP/IP adalah protokol yang digunakan oleh jaringan internet dan banyak jaringan lokal untuk bertukar informasi melalui jaringan. Protokol TCP/IP terdiri dari beberapa protokol yang saling terkait, termasuk IP (Internet Protocol), TCP (Transmission Control Protocol), dan UDP (User Datagram Protocol).

Berikut adalah beberapa contoh protokol TCP/IP:

  • HTTP (Hypertext Transfer Protocol) adalah protokol yang digunakan untuk mentransfer data (terutama halaman web) melalui internet. HTTP menggunakan TCP untuk mentransfer data secara terjamin dari server ke klien.
  • HTTPS (Hypertext Transfer Protocol Secure) adalah versi yang lebih aman dari HTTP. HTTPS menggunakan enkripsi untuk memastikan bahwa data yang dikirimkan tidak dapat diintip oleh pihak yang tidak memiliki otorisasi.
  • FTP (File Transfer Protocol) adalah protokol yang digunakan untuk mentransfer file antar host di dalam jaringan. FTP menggunakan TCP untuk mentransfer file secara terjamin dan terkontrol.
  • SMTP (Simple Mail Transfer Protocol) adalah protokol yang digunakan untuk mentransfer email antar server. SMTP menggunakan TCP untuk mentransfer email secara terjamin dan terkontrol.
  • Telnet adalah protokol yang digunakan untuk mengakses server jarak jauh melalui jaringan. Telnet menggunakan TCP untuk mentransfer data antar host.

Standards Organizations

Standards Organizations adalah lembaga atau organisasi yang bertanggung jawab untuk mengembangkan dan mengelola standar teknis di bidang teknologi informasi dan komunikasi. Mereka bertugas untuk menetapkan standar yang akan digunakan oleh perusahaan dan industri dalam pengembangan dan implementasi teknologi.

Standards Organizations memiliki peran yang sangat penting dalam menjaga agar teknologi tetap terintegrasi dan kompatibel satu sama lain. Tanpa standar yang jelas, teknologi mungkin tidak akan dapat bekerja sama dengan baik, atau bahkan tidak dapat bekerja sama sama sekali.

Beberapa contoh Standards Organizations yang terkenal di bidang teknologi informasi dan komunikasi adalah Institute of Electrical and Electronics Engineers (IEEE), International Organization for Standardization (ISO), dan Internet Engineering Task Force (IETF). Mereka bertanggung jawab untuk mengembangkan standar di bidang jaringan komputer, sistem operasi, dan teknologi internet.

Reference Models

Reference models adalah model-model yang digunakan sebagai acuan dalam mengembangkan suatu jaringan. Di Cisco, terdapat beberapa reference models yang sering digunakan, yaitu:

  • OSI (Open Systems Interconnection) Reference Model: Model ini menjelaskan bagaimana suatu jaringan komputer dapat terhubung dengan jaringan lainnya secara terstruktur. OSI Reference Model terdiri dari 7 lapisan, yaitu lapisan fisik, lapisan data link, lapisan jaringan, lapisan transportasi, lapisan sesi, lapisan presentasi, dan lapisan aplikasi.
  • TCP/IP (Transmission Control Protocol/Internet Protocol) Reference Model: Model ini merupakan model yang paling umum digunakan saat ini, terutama dalam jaringan Internet. TCP/IP Reference Model terdiri dari 4 lapisan, yaitu lapisan aplikasi, lapisan transportasi, lapisan jaringan, dan lapisan fisik.
  • Cisco Hierarchical Network Design Model: Model ini merupakan model yang dikembangkan oleh Cisco untuk membantu dalam merancang jaringan yang efisien dan dapat diandalkan. Model ini terdiri dari 3 lapisan, yaitu lapisan access, lapisan distribution, dan lapisan core.

Model-model reference tersebut dapat digunakan sebagai acuan dalam merancang, mengimplementasikan, dan mengelola suatu jaringan Cisco.

Data Encapsulation

Data encapsulation adalah proses mengemas suatu data dalam format yang sesuai dengan standar yang telah ditentukan, sehingga data tersebut dapat dikirimkan melalui suatu jaringan. Pada jaringan Cisco, data encapsulation terjadi pada setiap lapisan OSI Reference Model.

Pada lapisan fisik (layer 1), data dikemas dalam bentuk sinyal elektromagnetik yang dikirimkan melalui kabel atau media nirkabel.

Pada lapisan data link (layer 2), data dikemas dalam bentuk paket yang disebut frame. Frame ini berisi informasi yang diperlukan untuk mengirim data dari satu perangkat ke perangkat lainnya, seperti alamat MAC (Media Access Control).

Pada lapisan jaringan (layer 3), data dikemas dalam bentuk paket yang disebut datagram. Datagram ini berisi informasi yang diperlukan untuk mengirim data dari satu jaringan ke jaringan lain, seperti alamat IP (Internet Protocol).

Pada lapisan transportasi (layer 4), data dikemas dalam bentuk segmen yang disebut segment. Segment ini berisi informasi yang diperlukan untuk mengirim data dari satu host ke host lain, seperti nomor port.

Pada lapisan sesi (layer 5), data dikemas dalam bentuk sesi yang disebut session. Session ini berisi informasi yang diperlukan untuk mengatur komunikasi antara dua host.

Pada lapisan presentasi (layer 6), data dikemas dalam bentuk presentasi yang disebut presentation. Presentation ini berisi informasi yang diperlukan untuk mengubah format data agar sesuai dengan standar yang ditentukan.

Pada lapisan aplikasi (layer 7), data dikemas dalam bentuk yang sesuai dengan aplikasi yang digunakan.

Setiap lapisan mengemas data dengan menambahkan header dan trailer yang berisi informasi yang diperlukan untuk mengirim data tersebut ke lapisan yang lebih tinggi atau lapisan yang lebih rendah. Proses ini disebut data encapsulation.

Data Access

Data access merupakan lapisan pertama dalam Cisco Hierarchical Network Design Model. Lapisan ini bertanggung jawab untuk menghubungkan perangkat-perangkat akhir (end devices) seperti komputer, printer, atau smartphone ke jaringan.

Perangkat yang biasa digunakan di lapisan ini antara lain:

  • Switch: Digunakan untuk menghubungkan perangkat-perangkat akhir ke jaringan. Switch menerima paket data dari perangkat-perangkat akhir dan mengirimkannya ke perangkat-perangkat lainnya di jaringan.
  • Router: Digunakan untuk menghubungkan jaringan-jaringan lokal (LAN) ke jaringan-jaringan yang lebih besar (seperti Internet). Router menerima paket data dari perangkat-perangkat akhir dan mengirimkannya ke jaringan yang lebih besar sesuai dengan alamat tujuan yang terdapat pada paket data tersebut.
  • Access point: Digunakan untuk menghubungkan perangkat-perangkat akhir ke jaringan melalui wireless (nirkabel). Access point menerima sinyal wireless dari perangkat-perangkat akhir dan mengirimkannya ke jaringan yang lebih besar.
  • Perangkat-perangkat di lapisan data access ini biasanya berada di dekat perangkat-perangkat akhir dan memiliki kecepatan transfer data yang tinggi. Tujuan dari lapisan ini adalah untuk menyediakan akses yang cepat dan handal bagi perangkat-perangkat akhir ke jaringan.
