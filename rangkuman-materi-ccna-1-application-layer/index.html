Rangkuman Materi CCNA-1: Application Layer

Ferry Ananda Febian

Jan 5, 2023 • 4 min read

Application Layer adalah lapisan teratas dari model jaringan OSI (Open Systems Interconnection). Lapisan ini merupakan lapisan yang berinteraksi dengan aplikasi pengguna dan menyediakan layanan yang dibutuhkan oleh aplikasi tersebut. Contohnya, jika Anda menggunakan peramban web untuk mengakses sebuah situs web, maka lapisan aplikasi akan bertanggung jawab untuk mengirimkan permintaan ke server web dan menerima tanggapan yang dikirim kembali.

Application, Presentation, and Session

Sub ini menjelaskan bagaimana fungsi lapisan aplikasi, lapisan presentasi, dan lapisan sesi bekerja sama untuk menyediakan layanan jaringan ke aplikasi pengguna akhir.

Lapisan aplikasi, lapisan presentasi, dan lapisan sesi bekerja sama untuk menyediakan layanan jaringan ke aplikasi pengguna akhir.

Lapisan aplikasi merupakan lapisan teratas dari model jaringan OSI yang bertanggung jawab untuk berinteraksi dengan aplikasi pengguna dan menyediakan layanan yang dibutuhkan oleh aplikasi tersebut. Contohnya, jika Anda menggunakan peramban web untuk mengakses sebuah situs web, maka lapisan aplikasi akan bertanggung jawab untuk mengirimkan permintaan ke server web dan menerima tanggapan yang dikirim kembali.

Lapisan presentasi merupakan lapisan yang bertanggung jawab untuk mengubah data yang dikirimkan oleh lapisan aplikasi ke dalam bentuk yang dapat diterima oleh lapisan jaringan di bawahnya. Misalnya, jika lapisan aplikasi mengirimkan data teks ke lapisan presentasi, maka lapisan presentasi akan mengubah data tersebut ke dalam bentuk biner sebelum dikirimkan ke lapisan jaringan.

Lapisan sesi merupakan lapisan yang bertanggung jawab untuk mengelola sesi komunikasi antara aplikasi di komputer yang berbeda melalui jaringan. Misalnya, ketika Anda mengirim email atau pesan teks, lapisan sesi akan bertanggung jawab untuk memastikan bahwa data tersebut terkirim dengan benar dan tepat waktu.

Peer-to-Peer

Sub ini menjelaskan bagaimana aplikasi pengguna akhir beroperasi dalam jaringan peer-to-peer.

Dalam jaringan peer-to-peer (P2P), aplikasi pengguna akhir bekerja dengan cara mengirimkan dan menerima data langsung ke dan dari komputer lain yang terhubung ke jaringan, tanpa melalui server sentral. Artinya, setiap komputer yang terhubung ke jaringan P2P akan berfungsi sebagai server dan client pada saat yang sama.

Contoh aplikasi P2P yang populer adalah aplikasi berbagi file seperti BitTorrent. Ketika seseorang mengunduh file melalui aplikasi ini, ia akan terhubung ke jaringan P2P dan mengirim permintaan ke komputer lain yang memiliki file yang diinginkan. Kemudian, komputer tersebut akan mengirimkan file kepada pengguna yang meminta, dan proses ini akan terus berlangsung sampai seluruh file terunduh.

Dalam jaringan P2P, aplikasi pengguna akhir dapat bekerja dengan efisien karena tidak perlu mengandalkan server sentral untuk mengirim dan menerima data. Namun, jaringan P2P juga memiliki beberapa kekurangan, seperti kemampuan untuk memproses jumlah data yang lebih sedikit dibandingkan jaringan dengan server sentral dan rawan terhadap serangan keamanan.

Web and Email Protocols

Sub ini menjelaskan cara kerja protokol web dan email.

Web protocols dan email protocols adalah standar yang digunakan untuk mengirim dan menerima data melalui jaringan.

Web protocols adalah standar yang digunakan untuk mengirim dan menerima data melalui jaringan internet. Protokol yang paling populer adalah Hypertext Transfer Protocol (HTTP), yang digunakan oleh peramban web untuk mengirim permintaan ke server web dan menerima tanggapan yang dikirim kembali. HTTP menggunakan sistem permintaan-tanggapan untuk berkomunikasi, di mana peramban web mengirimkan permintaan ke server web, dan server web mengirimkan tanggapan yang berisi data yang diminta.

Email protocols adalah standar yang digunakan untuk mengirim dan menerima email melalui jaringan. Protokol yang paling populer adalah Simple Mail Transfer Protocol (SMTP), yang digunakan untuk mengirim email dari klien email ke server email, dan Post Office Protocol (POP) atau Internet Message Access Protocol (IMAP), yang digunakan untuk mengambil email dari server email dan menampilkannya di klien email.

IP Addressing Services

Sub ini menjelaskan cara kerja DNS dan DHCP.

Domain Name System (DNS) adalah jaringan global yang bertanggung jawab untuk mengkonversi alamat domain ke alamat IP. Alamat IP adalah nomor unik yang ditetapkan untuk setiap perangkat yang terhubung ke internet, sedangkan alamat domain adalah nama yang mudah diingat yang digunakan untuk mengidentifikasi perangkat tersebut.

Contohnya, ketika Anda mengetikkan “www.example.com” ke peramban web Anda, peramban web akan mengirim permintaan ke server DNS untuk mengkonversi alamat domain “www.example.com” ke alamat IP. Setelah alamat IP ditemukan, peramban web akan mengirim permintaan ke alamat IP tersebut untuk mengambil data dari server web yang terkait dengan alamat domain “www.example.com”.

Dynamic Host Configuration Protocol (DHCP) adalah protokol yang digunakan untuk mengatur konfigurasi jaringan secara otomatis. Ketika perangkat baru terhubung ke jaringan, DHCP akan memberikan perangkat tersebut alamat IP yang unik, subnet mask, gateway, dan informasi konfigurasi jaringan lainnya yang dibutuhkan.

DHCP bekerja dengan cara mengirimkan pesan ke perangkat yang meminta konfigurasi jaringan, yang disebut pesan “DHCP Discover”. Kemudian, server DHCP akan mengirimkan pesan “DHCP Offer” yang berisi informasi konfigurasi jaringan yang tersedia. Setelah perangkat menerima pesan “DHCP Offer”, ia akan mengirimkan pesan “DHCP Request” untuk meminta konfigurasi jaringan yang ditawarkan. Terakhir, server DHCP akan mengirimkan pesan “DHCP Ack” untuk mengonfirmasi bahwa konfigurasi jaringan telah diterapkan pada perangkat tersebut.

File Sharing Services

Sub ini menjelaskan bagaimana protokol transfer file beroperasi.

File Transfer Protocol (FTP) adalah sebuah protokol yang digunakan untuk mentransfer file antara komputer. FTP memungkinkan kita untuk mengirim atau mengunduh file dari server ke komputer atau sebaliknya. FTP menggunakan koneksi jaringan untuk mentransfer file, seperti internet atau jaringan lokal.

Untuk menggunakan FTP, kita perlu menggunakan aplikasi khusus atau client FTP. Client FTP akan terhubung ke server FTP menggunakan alamat IP atau nama host dan akan meminta kita untuk mengisi username dan password untuk mengakses server. Setelah terhubung, kita dapat mengirim atau mengunduh file dari server menggunakan client FTP.

FTP juga memiliki fitur keamanan tambahan dengan menggunakan protokol enkripsi, seperti Secure FTP (SFTP) atau FTP Secure (FTPS). Ini memungkinkan kita untuk mentransfer file dengan aman melalui jaringan yang tidak terjamin.
