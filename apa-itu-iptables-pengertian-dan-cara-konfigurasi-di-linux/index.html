Apa Itu Iptables: Pengertian dan Cara Konfigurasi di Linux

Ferry Ananda Febian

Nov 1, 2022 • 3 min read

Iptables adalah firewall yang sangat populer di linux, Iptables mengizinkan pengguna untuk memonitoring jaringan yang masuk dan keluar pada sistem komputer dengan rules atau aturan.

Iptables juga mengizinkan pengguna untuk mengkonfigurasi izin akses kontrol paket jaringan ke server pengguna. Di artikel ini kita belajar lebih banyak tentang Iptables, mulai dari mengkonfigurasi, mengaktifkan, menonaktifkan dan mengubah konfigurasi iptables.

Cara konfigurasi Iptables di Linux

1. Install Iptables

Buka terminal, jalankan perintah update package dan install iptables

sudo apt-get update;sudo apt-get install iptables

Check status iptables

sudo iptables -L -v

Anda akan melihat output berupa list dari rules iptables. Secara default, iptables belum mempunyai rules apapun.

Chain INPUT (policy ACCEPT 0 packets, 0 bytes) pkts bytes target prot opt in out source destination
Chain FORWARD (policy ACCEPT 0 packets, 0 bytes) pkts bytes target prot opt in out source destination
Chain OUTPUT (policy ACCEPT 0 packets, 0 bytes) pkts bytes target prot opt in out source destination

2. Konfigurasi Iptables

Iptables dapat memonitoring traffic dari dan ke server Anda sesuai rules yang diterapkan, aturan ini disebut CHAINS. Ketika paket sesuai dengan rule, maka akan diarahkan ke target.

Table default di iptables terdiri dari 3 aturan:

  • ACCEPT – menerima paket ke atau dari sumber
  • DROP – tidak mengizinkan paket ke atau dari sumber
  • RETURN – mengembalikan paket ke sumber sebelumnya dan stop penerusan paket

Dan ini adalah aturan lain dari iptables:

  • INPUT – untuk paket masuk
  • FORWARD – meneruskan paket yang masuk
  • OUTPUT – untuk paket yang keluar

Ini adalah perintah utama untuk memodifikasi iptables

sudo iptables -A  -i  -p  -s  --dport  -j

Berikut ini adalah penjelasan opsi:

  • – A – indikasi iptables yang ingin dimodifikasi
  • chain – nama aturan, misal INPUT, FORWARD or OUTPUT
  • -i interface – interface yang digunakan untuk kontrol jaringan
  • -p protocol – protokol jaringan, misal tcp/udp
  • -s source – alamat ip sumber –dport – nomor port tujuan
  • -j target – nama target misal ACCEPT, DROP atau RETURN.

Ini mendefinisikan aksi yang digunakan untuk kasus paket yang sesuai rule.

3. Simpan perubahan iptables

Perubahan baru iptables membutuhkan definisi ulang dengan reboot. Untuk membuat perubahan ini persisten, kita perlu menggunakan perintah berikut:

sudo /sbin/iptables-save > /etc/iptables.conf

Perintah diatas akan menyimpan konfigurasi rule iptables di path (/etc/iptables.conf). Anda perlu menjalankan perintah di atas setiap kali Anda membuat perubahan pada iptables.

Tambahkan perintah berikut ke /etc/rc.local untuk memuat ruleiptables yang disimpan pada saat reboot.

sudo iptables-restore < /etc/iptables.conf

Studi Kasus Iptables

Sekarang mari kita lihat beberapa contoh kasus iptables dibawah ini

1. Mengizinkan traffic dari localhost

sudo iptables -A INPUT -i lo -j ACCEPT

2. Mengizinkan traffic ke http, https dan port ssh

Berikut adalah aturan iptables untuk mengizinkan traffic ke http (port 80), https (port 443) dan ssh (port 22)

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

3. Mengizinkan traffic dari alamat IP spesifik

Berikut adalah rule iptables untuk mengizinkan traffic dari alamat IP yang spesifik misal 54.43.32.21. Ganti IP tersebut sesuai kebutuhan Anda.

sudo iptables -A INPUT -s 54.43.32.21 -j ACCEPT

4. Block traffic dari alamat IP spesifik

Berikut adalah rule iptables untuk tidak mengizinkan traffic dari alamat IP misal 54.43.32.21, Ganti IP tersebut sesuai kebutuhan Anda.

sudo iptables -A INPUT -s 54.43.32.21 -j DROP

5. Drop traffic ke semua port

Berikut adalah rule iptables untuk mendrop traffic ke semua port selain iptables yang telah ditentukan.

sudo iptables -A INPUT -j DROP

6. Tampilkan semua daftar rules iptables

Berikut adalah perintah untuk menampilkan semua rules iptables dengan opsi -L

sudo iptables -L

7. Hapus semua daftar rules iptables

Berikut adalah perintah untuk menghapus semua rules iptables dengan opsi -F

sudo iptables -F

8. Hapus spesifik rule iptables

Jika Anda ingin menghapus rule IP tables spesifik, gunakan opsi -D, spesifik nama nama CHAIN dan nomor rule. Berikut adalah contoh perintah hapus spesifik rule iptables misal pada rule nomor 5.

sudo iptables -D INPUT 5

9. Simpan perubahan iptables

Perubahan baru iptables membutuhkan definisi ulang dengan reboot. Untuk membuat perubahan ini persisten, kita perlu menggunakan perintah berikut:

sudo /sbin/iptables-save > /etc/iptables.conf

Perintah diatas akan menyimpan konfigurasi rule iptables di path (/etc/iptables.conf). Anda perlu menjalankan perintah di atas setiap kali Anda membuat perubahan pada iptables.

Tambahkan perintah berikut ke /etc/rc.local untuk memuat ruleiptables yang disimpan pada saat reboot.

sudo iptables-restore < /etc/iptables.conf

10. Disable iptables pada sistem

sudo iptables -F  # sudo /sbin/iptables-save

Oke selesai, seperti itu cara menggunakan iptables untuk melindungi server Anda dari traffic yang tidak dikenal.
