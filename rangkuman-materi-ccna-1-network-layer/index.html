Rangkuman Materi CCNA-1: Network Layer

Ferry Ananda Febian

Jan 4, 2023 • 3 min read

Network Layer Characteristics

Lapisan jaringan (network layer) pada model OSI (Open Systems Interconnection) menggunakan protokol IP (Internet Protocol) untuk mentransmisikan data melalui jaringan dengan benar. Protokol IP ini memiliki beberapa kelebihan yang membuatnya dapat digunakan untuk komunikasi yang dapat diandalkan, diantaranya adalah:

  • Addressing: Protokol IP menggunakan alamat IPv4 (Internet Protocol version 4) atau IPv6 (Internet Protocol version 6) untuk mengidentifikasi perangkat jaringan. Alamat ini terdiri dari empat atau enam blok angka yang dipisahkan oleh titik (.) dan menggunakan notasi desimal. Alamat ini harus unik dan tidak boleh sama dengan alamat dari perangkat jaringan lainnya, sehingga data dapat dikirim dan diterima dengan benar.
  • Routing: Protokol IP menggunakan tabel routing untuk menentukan jalur terbaik untuk mentransmisikan data melalui jaringan. Tabel ini disimpan pada router yang menghubungkan beberapa jaringan dan mengatur lalu lintas data. Dengan tabel routing ini, data dapat dikirim ke perangkat tujuan dengan efisien dan cepat.
  • Error handling: Protokol IP menggunakan mekanisme error handling untuk memastikan bahwa data yang dikirimkan dapat diterima dengan benar. Mekanisme ini menggunakan teknik-teknik seperti checksum dan parity bit untuk mengecek integritas data. 

IPv4 Packet

Bidang utama dalam paket IPv4 adalah:

  • Versi: Bidang ini menunjukkan versi protokol IPv4 yang sedang digunakan.
  • Panjang Header: Bidang ini menentukan panjang header dalam 32-bit kata.
  • Tipe Layanan: Bidang ini memungkinkan pengirim untuk menentukan tipe layanan yang harus diterima oleh paket.
  • Panjang Total: Bidang ini menentukan panjang total paket, termasuk header dan data.
  • Identifikasi: Bidang ini digunakan untuk mengidentifikasi paket dan membantu proses fragmentasi dan reassembling.
  • Flags: Bidang ini digunakan untuk mengontrol fragmentasi dan reassembling paket.
  • Offset Fragmentasi: Bidang ini menentukan offset data dalam paket, terhadap awal paket asli yang tidak terfragmentasi.
  • Time to Live: Bidang ini menentukan jumlah maksimum hops yang dapat dilakukan oleh paket sebelum dibuang.
  • Protokol: Bidang ini menentukan protokol lapisan atas yang terdapat dalam bidang data paket.
  • Header Checksum: Bidang ini digunakan untuk mendeteksi error dalam header.
  • Alamat Sumber: Bidang ini menentukan alamat IPv4 pengirim.
  • Alamat Tujuan: Bidang ini menentukan alamat IPv4 penerima yang dituju.

Bidang-bidang header ini digunakan oleh router untuk mengirimkan paket ke arah tujuannya.

IPv6 Packet

Di dalam paket IPv6, terdapat beberapa field header utama yang memiliki peran penting dalam proses transmisi data. Berikut ini adalah uraian singkat tentang masing-masing field tersebut:

  • Version: Field ini menunjukkan versi protokol IPv6 yang sedang digunakan.
  • Traffic Class: Field ini digunakan untuk mengategorikan paket data berdasarkan tingkat prioritasnya.
  • Flow Label: Field ini digunakan untuk menandai paket data yang terkait dengan suatu aliran data tertentu, sehingga dapat diidentifikasi dengan mudah.
  • Payload Length: Field ini menunjukkan panjang payload (isi) dari paket data.
  • Next Header: Field ini menunjukkan tipe header selanjutnya yang terdapat dalam paket data.
  • Hop Limit: Field ini menunjukkan jumlah maksimal hop yang diizinkan untuk paket data tersebut selama proses transmisi.
  • Source Address: Field ini menyimpan alamat IP sumber paket data.
  • Destination Address: Field ini menyimpan alamat IP tujuan paket data.

How a Host Routes

Perangkat jaringan menggunakan tabel routing untuk mengarahkan paket ke jaringan tujuan dengan cara mencari alamat tujuan yang tercantum dalam paket dan membandingkannya dengan entri di dalam tabel routing. Apabila terdapat match, maka perangkat akan menentukan interface dan rute terbaik yang akan digunakan untuk mengirimkan paket ke tujuan.

Setiap perangkat jaringan memiliki tabel routing sendiri, yang dapat berisi informasi rute default serta entri rute yang ditambahkan secara manual atau dinamis melalui protokol routing seperti OSPF atau BGP. Ketika paket datang ke perangkat, perangkat tersebut akan memeriksa tujuan yang tercantum dalam paket dan mencari entri di tabel routing yang sesuai. Apabila ditemukan entri yang sesuai, perangkat akan mengarahkan paket ke jaringan tujuan melalui rute yang telah ditentukan. Apabila tidak ditemukan entri yang sesuai, perangkat akan mengarahkan paket ke jaringan default yang telah ditentukan sebelumnya.

Secara garis besar, proses pencarian rute di tabel routing terdiri dari beberapa tahap, yaitu:

  • Pencarian alamat tujuan dalam paket data.
  • Pencocokan alamat tujuan dengan entri di tabel routing.
  • Pemilihan rute terbaik berdasarkan kriteria yang telah ditentukan (misalnya, jumlah hop atau kecepatan transmisi).
  • Pengiriman paket melalui rute yang telah dipilih.

Dengan menggunakan tabel routing, perangkat jaringan dapat dengan mudah menentukan rute terbaik untuk mengirimkan paket ke tujuan, sehingga proses transmisi data dapat berlangsung dengan efisien.

Router Routing Tables

Tabel routing adalah daftar yang digunakan oleh router untuk menentukan rute terbaik yang akan digunakan untuk mengirimkan paket data ke tujuan. 

Contoh output IPv4 Routing Table

Setiap entri di dalam tabel routing terdiri dari beberapa field yang memiliki fungsi masing-masing. Berikut ini adalah uraian singkat tentang masing-masing field tersebut:

  • Destination: Field ini menyimpan alamat tujuan yang akan dituju oleh paket data.
  • Next Hop: Field ini menyimpan alamat perangkat jaringan (misalnya, router) yang akan menerima paket data selanjutnya.
  • Interface: Field ini menyimpan interface (port) yang akan digunakan router untuk mengirimkan paket data ke perangkat jaringan selanjutnya.
  • Metric: Field ini menyimpan informasi tentang biaya atau jarak yang harus ditempuh untuk mencapai tujuan, yang dapat digunakan oleh router untuk menentukan rute terbaik.
  • Administrative Distance: Field ini menyimpan nilai yang menunjukkan kepercayaan terhadap sumber informasi rute. Nilai yang lebih rendah menunjukkan bahwa sumber informasi tersebut dianggap lebih dapat dipercaya.
  • Timer: Field ini menyimpan informasi tentang berapa lama entri tersebut akan tetap valid sebelum harus diperbaharui.
  • Protocol: Field ini menyimpan informasi tentang protokol routing yang digunakan untuk mengumpulkan informasi rute yang tercantum di dalam tabel.

Dengan menggunakan field-field tersebut, router dapat dengan mudah menentukan rute terbaik yang akan digunakan untuk mengirimkan paket data ke tujuan.
