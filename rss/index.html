List sangat penting dalam hal mengembangkan UI situs web. Terutama sering digunakan untuk menampilkan menu di situs web, misalnya menu navbar. Pada JavaScript biasa, kita dapat menggunakan array untuk membuat list. Kita dapat membuat list di React dengan cara yang sama seperti yang dilakukan pada javascript biasa.

Selanjutnya artikel ini akan membahas lebih dalam terkait list pada React.

Pengertian

List pada React adalah salah satu tipe data yang digunakan untuk menyimpan kumpulan nilai atau elemen dalam sebuah variabel. List juga bisa disebut sebagai daftar atau array.

Cara Membuat List

Untuk membuat list pada React, kita dapat menggunakan sintaksis yang sama seperti di JavaScript, yaitu dengan menempatkan elemen-elemen atau nilai-nilai yang ingin kita simpan di dalam tanda kurung siku ([]). Kita juga dapat menggunakan method-method seperti map(), filter(), dan reduce() untuk memanipulasi data di dalam list tersebut.

Berikut adalah contoh fungsi JavaScript map() untuk menelusuri list:

var nomor = [1, 2, 3, 4, 5];
const updateNomor = nomor.map((number) => {
  return number + 2;
});

console.log(updateNomor);
JavaScript

Kode diatas akan menghasilkan output seperti di bawah ini:

[3, 4, 5, 6, 7]

Selanjutnya kita akan coba menampilkan kumpulan data dalam bentuk list atau daftar di dalam komponen React. Dalam React, rendering list biasanya dilakukan dengan menggunakan konsep “pengulangan” atau “looping”, yang memungkinkan kita untuk menampilkan setiap elemen dalam list secara otomatis.

import React from "react";
import ReactDOM from "react-dom";

function Navmenu(props) {
  const list = props.menuitems;
  const updatedList = list.map((listItems) => {
    return <li>{listItems}</li>;
  });
  return <ul>{updatedList}</ul>;
}

const menuItems = [1, 2, 3, 4, 5];

ReactDOM.render(<Navmenu menuitems={menuItems} />, document.getElementById("root"));
JavaScript

Output yang dihasilkan:

Bisa dilihat pada output di atas, unordered list berhasil di render ke browser tetapi pesan warning dicatat ke konsol.

Peringatan: Setiap child dalam larik atau iterator harus memiliki penyangga “key” yang unik

Pesan warning di atas mengatakan bahwa setiap item daftar dalam daftar unordered harus memiliki kunci yang unik. “Kunci” adalah atribut string khusus yang perlu disertakan saat membuat daftar elemen di React. Materi tentang key akan dibahas secara rinci di artikel selanjutnya.

Berikut adalah contoh kode React yang sudah diperbaharui dengan key:

import React from "react";
import ReactDOM from "react-dom";

function Navmenu(props) {
  const list = props.menuitems;
  const updatedList = list.map((listItems) => {
    return <li key={listItems.toString()}>{listItems}</li>;
  });

  return <ul>{updatedList}</ul>;
}

const menuItems = [1, 2, 3, 4, 5];

ReactDOM.render(<Navmenu menuitems={menuItems} />, document.getElementById("root"));
JavaScript

Kode diatas akan memberikan output yang sama dengan kode sebelumnya tetapi kali ini tanpa peringatan apapun. Key digunakan dalam React untuk mengidentifikasi item mana dalam daftar yang diubah, diperbarui, atau dihapus. Dengan kata lain, key dapat digunakan untuk memberi identitas pada elemen dalam daftar.

Di artikel sebelumnya yaitu Pengantar JSX, kita sudah mempelajari bahwa developer bisa menggunakan JSX untuk menyimpan markup HTML di dalam variabel Javascript. Karena, ReactJS adalah sebuah library untuk membangun Antarmuka Pengguna yang aktif sehingga rendering adalah salah satu bagian yang tidak terpisahkan dari ReactJS. React menyediakan sebuah paket yang bernama ReactDOM kepada para pengembang untuk mengakses dan memodifikasi DOM. Mari kita simak lebih lanjut.

Pengertian DOM

DOM (Document Object Model) adalah representasi dari struktur HTML dari suatu halaman web yang dapat dimanipulasi menggunakan JavaScript. Pada React, ReactDOM adalah library JavaScript yang memungkinkan kita untuk memanipulasi dan mengupdate DOM secara efisien. ReactDOM menyediakan metode dan fungsi-fungsi untuk mengakses elemen DOM, merender komponen React ke dalam DOM, dan mengupdate elemen DOM dengan data yang baru.

Secara spesifik, ReactDOM.render() adalah fungsi utama yang digunakan untuk merender sebuah komponen React ke dalam DOM. Fungsi ini menerima dua parameter: elemen React yang ingin dirender dan elemen DOM tempat dimana elemen React tersebut akan dirender. Setelah elemen React di-render ke dalam DOM, ReactDOM memastikan bahwa setiap perubahan pada elemen React akan diupdate ke dalam DOM secara otomatis.

Pengertian ReactDOM

ReactDOM adalah sebuah paket yang menyediakan metode-metode khusus DOM yang dapat digunakan pada aplikasi web untuk memungkinkan cara yang efisien dalam mengelola elemen-elemen DOM pada halaman web. Berikut adalah API metode-metode yang umum digunakan dalam ReactDOM: 

  • render()
  • findDOMNode()
  • unmountComponentAtNode()
  • hydrate()
  • createPortal()

Prasyarat: Untuk menggunakan ReactDOM di aplikasi web React, kita harus mengimpor ReactDOM terlebih dahulu dari paket react-dom dengan menggunakan perintah berikut:  

import ReactDOM from 'react-dom'

render() Function

Fungsi ini umumnya digunakan untuk mendapatkan node DOM dimana komponen React tertentu di-render. Cara ini sangat jarang digunakan seperti contoh berikut yang dapat dilakukan dengan menambahkan atribut ref ke setiap komponen itu sendiri. 

Sintaks: 

ReactDOM.findDOMNode(component)

Parameter: Metode ini mengambil satu parameter komponen yang mengharapkan sebuah Komponen React untuk dicari di DOM Browser. 

Tipe Return: Fungsi ini mengembalikan simpul DOM, tempat komponen di-render jika berhasil, jika tidak, maka akan bernilai nol.

unmountComponentAtNode() Function

Fungsi ini digunakan untuk menghapus sebuah komponen React yang telah dirender ke dalam sebuah elemen DOM. Fungsi ini menerima satu parameter, yaitu elemen DOM tempat di mana komponen React telah dirender. Setelah fungsi ini dipanggil, komponen React akan dihapus dari elemen DOM tersebut.

Sintaks: 

ReactDOM.unmountComponentAtNode(container)

Parameter: Metode ini mengambil sebuah kontainer parameter tunggal yang mengharapkan kontainer DOM dari komponen React yang harus dihapus.

Tipe Return: Tipe kembalian dari fungsi ini adalah sebuah boolean, yaitu true jika komponen berhasil dihapus dari elemen DOM, dan false jika tidak ada komponen yang dihapus karena elemen DOM tersebut tidak memiliki komponen yang dirender.

hydrate() Function

Fungsi ini digunakan untuk memasangkan tampilan yang sudah di render pada server dengan JavaScript yang sudah dimuat pada sisi klien. Fungsi ini berguna ketika kita ingin meningkatkan performa aplikasi web dengan mengurangi waktu yang dibutuhkan oleh React untuk merender ulang tampilan yang sudah ada.

Sintaks:

ReactDOM.hydrate(element, container, callback)

Parameter: 

  • element (elemen React): Elemen React yang ingin dirender.
  • container (elemen DOM): Elemen DOM tempat dimana elemen React akan dirender.
  • callback (fungsi opsional): Fungsi yang akan dijalankan setelah proses hydrate selesai.

Tipe Return: Fungsi ini mencoba melampirkan event listener ke markup yang ada dan mengembalikan referensi ke komponen atau null jika komponen tanpa nama di-render.

createPortal() Function

Fungsi yang memungkinkan untuk merender elemen React di luar hierarki DOM utama, yaitu di tempat lain pada dokumen HTML. Dengan menggunakan createPortal(), maka kita dapat merender elemen React di dalam kontainer yang terpisah dari elemen DOM di mana komponen utama dirender.

Sintaks:

ReactDOM.createPortal(child, container)

Parameter:

  • child: Elemen React yang ingin dirender.
  • container: DOM element di mana elemen React akan dirender.

Tipe Return: Fungsi ini tidak mengembalikan apapun.

Poin Penting

  • ReactDOM.render() menggantikan child dari kontainer yang diberikan jika ada. Fungsi ini menggunakan algoritma diff yang sangat efisien dan dapat memodifikasi subtree dari DOM.
  • Fungsi findDOMNode() hanya dapat diimplementasikan pada komponen yang sudah terpasang sehingga komponen fungsional tidak dapat digunakan dalam metode findDOMNode().
  • ReactDOM menggunakan observable sehingga menyediakan cara yang efisien untuk menangani DOM.
  • ReactDOM dapat digunakan di sisi klien dan server.

Virtual DOM adalah konsep yang mendasari cara kerja React dalam merender tampilan pada aplikasi web. Virtual DOM adalah sebuah representasi virtual dari DOM (Document Object Model) yang berisi struktur tampilan (HTML) dan data pada aplikasi. Virtual DOM adalah teknologi yang memungkinkan React untuk mempercepat proses rendering tampilan pada aplikasi web, dengan mengurangi jumlah perubahan pada DOM yang harus dilakukan oleh browser.

Ketika ada perubahan pada data di aplikasi, React akan melakukan perbandingan antara Virtual DOM baru dan Virtual DOM sebelumnya. Setelah dilakukan perbandingan, React akan mengidentifikasi perubahan-perubahan yang terjadi pada Virtual DOM dan menerapkannya pada DOM asli. Hal ini memungkinkan React untuk mengurangi jumlah manipulasi DOM yang dilakukan pada browser, yang mana manipulasi DOM yang terlalu banyak dapat menyebabkan aplikasi menjadi lambat dan tidak responsif.

contoh : 

import React, { useState } from 'react';

function Example() {
  const [count, setCount] = useState(0);
  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={handleClick}>Klik saya</button>
    </div>
  );
}

Pada contoh diatas, kode menggunakan fungsi useState untuk menyimpan nilai state count dan fungsi setCount untuk mengubah nilai state tersebut. Ketika tombol “Klik saya” di klik, maka nilai state count akan dinaikkan sebesar 1 dan komponen akan di-render ulang.

Perhatikan bahwa pada saat state diubah, React akan membuat sebuah Virtual DOM baru yang memperbarui nilai dari state count. Setelah itu, React akan membandingkan Virtual DOM baru dengan Virtual DOM sebelumnya, dan mengupdate DOM asli hanya pada elemen yang berubah. Hal ini memungkinkan React untuk menghindari rendering ulang keseluruhan tampilan pada aplikasi dan mempercepat kinerja aplikasi secara signifikan.

Di materi sebelumnya, kita sudah belajar tentang JSX dan di artikel ini kita akan lanjut membahas lebih dalam tentang babel.

Pengertian Babel

Babel adalah sebuah transpiler atau alat pengubah kode yang digunakan untuk mengubah kode JavaScript modern menjadi kode JavaScript yang lebih kompatibel dengan browser atau lingkungan runtime yang lebih lama.

Babel biasanya digunakan untuk mengubah kode JavaScript yang ditulis dengan fitur-fitur terbaru (seperti arrow function, template literals, dan destructuring assignment) menjadi kode JavaScript yang dapat dijalankan pada browser atau lingkungan runtime yang tidak mendukung fitur-fitur tersebut. Dengan menggunakan Babel, kita dapat menulis kode JavaScript yang lebih baru dan modern tanpa khawatir tentang kompatibilitas lintas-platform.

Babel juga dapat digunakan untuk mengubah kode JavaScript dari bahasa lain (seperti TypeScript atau Flow) menjadi kode JavaScript biasa yang dapat dijalankan di lingkungan yang berbeda. Babel dapat diinstal dan digunakan melalui paket npm dan memiliki berbagai plugin yang dapat ditambahkan untuk mengubah kode JavaScript dengan cara yang berbeda-beda.

Contoh sederhana

Berikut adalah contoh sederhana dari versi standar terbaru ES 2017 dan mari kita lihat apa yang terjadi pada kode saat kita meneruskannya ke babel.

// contoh kode javascript versi baru
const var1 = (x) => {x*2};

const a = () => {};

const b = (x) => x;

[1, 2, 3].map((n)=> n+1);

Beberapa kode di atas tidak didukung di beberapa browser, jadi setelah melakukan transpiler melalui Babel kita akan mendapatkan kode seperti berikut:

// setelah transpiler
"use strict";

var var1 = function var1(x) {
  x * 2;
};

var a = function a() {};

var b = function b(x) {
  return x;
};

[1, 2, 3].map(function (n) {
  return n + 1;
});

Sebelumnya kita sudah tau kalau React itu bukan merupakan framework, melainkan library JavaScript untuk membangun antarmuka aplikasi. Meskipun dibangun dengan JavaScript, tetapi React tidak dapat ditulis dengan JavaScript secara langsung, melainkan harus ditulis dalam bentuk JSX. Apa itu JSX? Kita akan bahas lebih jauh tentang JSX dalam artikel ini.

Contoh kode JSX:

const contoh = <h1>Ini adalah contoh JSX</h1>;

Kode diatas sekilas mirip seperti HTML dan variable JavaScript, kode diatas disebut dengan JSX. JSX adalah sintaksis dari JavaScript biasa dan digunakan untuk membuat elemen React. Elemen-elemen ini kemudian di-render ke React DOM.

JSX Template

JSX lebih cepat daripada JavaScript biasa karena melakukan pengoptimalan saat kode diterjemahkan ke JavaScript biasa. Hal ini memudahkan developer untuk membuat template, dibandingkan dengan cara tradisional yang harus memisahkan markup dan logika dalam file terpisah.

React dapat menggunakan ekspresi JavaScript biasa dengan JSX. Untuk menampilkan ekspresi JavaScript dalam kode yang ditulis dalam JSX, kita harus membungkus ekspresi tersebut dalam kurung kurawal {}. Perhatikan contoh berikut ini:

import React from 'react';
import ReactDOM from 'react-dom';

const name = "unixkuy";
const element = <h1>Belajar JSX,
{ name }.Selamat Datang di unixkuy.< /h1>;
ReactDOM.render(
    element,
    document.getElementById("root")
);

Pada program di atas, kita telah menyematkan ekspresi javascript const name = “unixkuy”; pada kode JSX. Ada beberapa baris di bagian atas yang digunakan untuk mengimpor beberapa API React yang akan dijelaskan di artikel selanjutnya. Selain itu kita menyematkan penggunaan ekspresi JavaScript ke dalam JSX dengan membungkusnya dalam tanda kurung kurawal kecuali pernyataan if-else. Tetapi kita dapat menggunakan pernyataan kondisional ini sebagai pengganti pernyataan if-else di JSX. 

Biar lebih jelas, perhatikan kode dibawah ini, dimana ekspresi bersyarat disematkan dalam JSX:

import React from 'react';
import ReactDOM from 'react-dom';

let i = 1;
const element = <h1>{ (i == 1) ? 'Belajar React Otodidak!' : 'False!' } < /h1>;

ReactDOM.render(
element,
document.getElementById("root")
);

Pada contoh diatas, variabel i akan diperiksa apakah bernilai 1. Karena nilainya sama dengan 1 maka string ‘Belajar React Otodidak!’ akan dikembalikan ke kode JSX. Jika kita mengubah nilai variabel i maka string ‘False’ akan dikembalikan.

Atribut dalam JSX

JSX memungkinkan kita untuk menggunakan atribut dengan elemen HTML seperti yang kita lakukan pada HTML biasa. Tetapi pada JSX, penamaan class menggunakan camelCase. Sebagai contoh, class pada HTML menjadi namaClass pada JSX. Alasannya karena beberapa nama atribut dalam HTML seperti ‘class’ adalah kata kunci yang dicadangkan dalam JavaScripts. Jadi, untuk menghindari masalah ini, JSX menggunakan konvensi penamaan huruf besar-kecil untuk atribut. Kita juga dapat menggunakan atribut khusus di JSX. Nama atribut tersebut harus diawali dengan data-. Pada contoh dibawah ini, kita telah menggunakan atribut custom dengan nama data-sampleAttribute untuk tag <h2>.

contoh : 

import React from 'react';
import ReactDOM from 'react-dom';

const element = <div><h1 className = "hello">Hello Geek</h1>
            <h2 data-sampleAttribute="sample">Custom attribute</h2>< /div>;
ReactDOM.render(
    element,
    document.getElementById("root")
);

Menentukan nilai atribut

JSX memungkinkan kita untuk menentukan nilai atribut dengan dua cara: 

1.String literal: Kita dapat menentukan nilai atribut sebagai string yang dikodekan dengan menggunakan tanda kutip: 

const var = <h1 className = "firstAttribute">Hello!</h1>;

2. Sebagai ekspresi: Kita bisa menentukan atribut sebagai ekspresi menggunakan kurung kurawal {}: 

const var = <h1 className = {varName}>Hello!</h1>;

Membungkus elemen atau Anak di JSX

Maksudnya apa bang? membungkus elemen atau anak di JSX berarti menempatkan satu atau beberapa elemen JSX di dalam elemen lain sebagai child atau anaknya. Hal ini sering digunakan dalam pembuatan tampilan di React untuk mengatur struktur dan hirarki dari elemen-elemen JSX.

Contohnya, jika kita ingin membuat komponen tombol dengan teks di dalamnya, kita dapat menggunakan sintaksis JSX untuk menempatkan elemen teks sebagai child atau anak dari elemen tombol, seperti contoh berikut:

<button>
Klik Tombol!
</button>

Di sini, elemen teks “Klik Tombol!” menjadi child dari elemen tombol. Kita juga dapat menempatkan lebih dari satu elemen sebagai child di dalam elemen lain, seperti contoh berikut:

<div>
<h1>Hello, world!</h1>
  <p>Welcome to my website.</p>
</div>

Dapat dilihat, elemen h1 dan p menjadi child dari elemen div. Dengan menggunakan teknik ini, kita dapat membuat tampilan yang lebih kompleks dengan mengatur hirarki dari elemen-elemen JSX yang berbeda.

Pengertian Linux

Linux adalah sistem operasi yang juga termasuk kedalam keluarga Unix. Linux merupakan sistem operasi yang bersifat open source, artinya kode sumbernya bisa dilihat dan dimodifikasi oleh siapapun.

Sejarah Linux

Sejarah sistem operasi Linux dimulai pada tahun 1991, ketika seorang mahasiswa Finlandia bernama Linus Torvalds yang kini kita kenal sebagai bapaknya Linux, Linus ini mengembangkan sistem operasi yang kemudian dikenal dengan nama Linux, mirip dengan namanya sendiri. Pada awalnya, Linus Torvalds hanya ingin membuat sistem operasi yang sederhana untuk digunakan pada komputer pribadi nya. Namun, kemudian ia memutuskan untuk membagikan kode sumbernya secara gratis kepada orang lain melalui internet, sehingga orang lain juga dapat ikut berkontribusi dalam pengembangan Linux.

Dari sini, Linux mulai dikembangkan oleh banyak orang di seluruh dunia dan banyak distribusi atau juga disebut versi, kalau bahasa gaulnya itu distro, maknanya sama aja, yaitu versi Linux yang dikembangkan oleh berbagai perusahaan dan komunitas. Linux kini menjadi sistem operasi populer karena dapat diinstall pada berbagai macam perangkat, memiliki keamanan yang tinggi dan juga bersifat open source, sehingga kode sumbernya bisa dimodifikasi oleh siapapun.

Linux saat ini telah menjadi salah satu sistem operasi yang paling banyak digunakan di seluruh dunia, terutama pada server, superkomputer, dan sistem embedded (perangkat yang terintegrasi dengan sistem operasi).

Kelebihan Sistem Operasi Linux

Selanjutnya kita akan bahas, apa aja sih kelebihan sistem operasi Linux ini.

Berikut ini adalah beberapa kelebihan sistem operasi Linux:

  • Gratis dan open source: Kita bisa mendapatkan Linux secara cuma-cuma dan bisa melihat, mengubah, dan membagikan kode sumbernya.
  • Performa yang baik: Linux banyak digunakan pada server karena performanya yang baik. Sistem operasi ini juga bisa berjalan dengan baik pada perangkat keras yang spesifikasinya rendah. Ini jadi salah satu alasan, kenapa senang menggunakan Linux.
  • Kemudahan instalasi: Proses instalasi Linux biasanya lebih mudah dibandingkan dengan sistem operasi lainnya. Kita bisa menginstal Linux dengan mudah pada berbagai macam perangkat.
  • Kustomisasi yang mudah: Kita bisa mengubah tampilan dan fitur Linux sesuai dengan kebutuhan da dan juga bisa memasang aplikasi tambahan yang tidak tersedia di dalam repositori resmi.
  • Keamanan yang tinggi: Linux memiliki keamanan yang tinggi karena tidak banyak terkena serangan virus dan malware. Selain itu, Linux juga memiliki fitur yang memungkinkan pengguna untuk membatasi akses pengguna ke sistem.
  • Dukungan komunitas yang luas: Linux memiliki komunitas pengguna yang sangat luas sehingga pengguna yang masih awam bisa mendapatkan bantuan dan dukungan dari para pengguna lainnya jika mengalami masalah.

Kekurangan Sistem Operasi Linux

Kesempurnaan hanya milik Tuhan, sistem operasi Linux juga buatan manusia dan tentu ada kekurangannya.

Berikut adalah beberapa kekurangan sistem operasi Linux:

  • Kurangnya dukungan aplikasi komersial: Beberapa aplikasi populer hanya tersedia untuk sistem operasi Windows, sehingga pengguna Linux mungkin harus mencari alternatif atau menggunakan aplikasi yang tidak sepenuhnya sesuai dengan kebutuhan mereka.
  • Kurangnya kemudahan penggunaan: Bagi pengguna yang tidak terbiasa dengan sistem operasi Linux, mungkin akan mengalami kesulitan dalam menggunakan dan mengatur sistem operasi ini.
  • Kurangnya dukungan hardware: Beberapa jenis hardware mungkin tidak didukung oleh Linux atau hanya didukung secara terbatas.
  • Kurangnya dokumentasi: Beberapa distribusi Linux mungkin memiliki dokumentasi yang kurang lengkap dibandingkan dengan sistem operasi lainnya.
  • Kurangnya dukungan teknis: Dukungan teknis untuk Linux mungkin tidak seluas dukungan untuk sistem operasi lain, terutama bagi pengguna yang tidak membeli lisensi dari perusahaan pengembang Linux.

Perintah berikut digunakan untuk menampilkan semua daftar parameter kernel linux dengan menambahkan opsi -a atau –all

Perintah berikut akan menampilkan semua daftar nama variable kernel dengan menambahkan opsi -N

sudo sysctl -a -N

sudo sysctl -a --deprecated

Perintah berikut akan menampilkan variable yang dicari beserta nilai yang terdapat pada variable tersebut.

sudo sysctl kernel.ostype

Perlu diketahui format penulisan variabe kernel Linux adalah sebagai berikut:

<nama>=<value>

sudo sysctl net.ipv4.tcp_fin_timeout=60

atau dengan menambahkan opsi -w dan operator >> (append).

sudo sysctl -w net.ipv4.tcp_fin_timeout=60 >> /etc/sysctl.conf

sysctl -w net.ipv4.tcp_fin_timeout=0

sysctl -w vm.swappiness=10

10. Reload sysctl.conf variable

Di Linux kita dapat mereload variable yang baru di modifikasi tanpa mereload perangkat dengan opsi –system

sudo sysctl  --system

Apakah Anda ingin mengetahui riwayat aplikasi yang pernah diinstal di sistem operasi Linux? Berikut adalah panduan sederhana yang akan membantu Anda melacak riwayat instalasi aplikasi di Linux.

Perintah yang digunakan untuk mengetahui riwayat aplikasi apa saja yang pernah diinstal adalah dengan memanfaatkan perintah history.

Perintah history pada Linux adalah perintah bawaan yang menampilkan daftar perintah yang pernah dijalankan dalam sesi terminal saat ini.

Untuk menampilkan daftar aplikasi apa saja yang pernah diinstal pada output history yaitu dengan menambahkan perintah grep dan dilanjut dengan “nilai string”. Pada kasus ini, kita bisa menggunakan perintah berikut ini:

history | grep “sudo apt install”

Penjelasan perintah diatas

  • history digunakan untuk menampilkan daftar perintah yang pernah dijalankan
  • | adalah tanda untuk menggabungkan perintah grep dengan perintah history
  • grep digunakan untuk mencari nilai string atau teks dari sebuah output
  • “sudo apt install” adalah nilai string yang ditetapkan
