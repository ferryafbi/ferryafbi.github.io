Key Based Authentication di SSH Linux: Pengertian dan Cara Mengaktifkan

Ferry Ananda Febian

Oct 28, 2022 • 3 min read

Key based authentication ssh adalah metode autentikasi yang menggunakan pasangan kunci publik-privat untuk mengidentifikasi pengguna yang ingin mengakses sebuah server. Kunci publik akan diberikan kepada server, sementara kunci privat akan disimpan oleh pengguna. Ketika pengguna ingin mengakses server, ia akan memasukkan kunci privatnya untuk memverifikasi identitasnya. Jika kunci privat terverifikasi, maka pengguna akan dapat mengakses server. Key based authentication ssh merupakan salah satu metode yang aman dan mudah digunakan untuk autentikasi pengguna pada server.

SSH mendukung berbagai jenis otentikasi seperti otentikasi berbasis kata sandi dan otentikasi berbasis kunci pribadi. Pada artikel ini, kita akan belajar cara mengaktifkan otentikasi berbasis kunci di SSH.

Cara Mengaktifkan Otentikasi Berbasis Kunci di SSH

Berikut adalah langkah-langkah untuk mengaktifkan otentikasi berbasis kunci di SSH.

1. Instal Open SSH

Anda memerlukan openssh-client pada sistem operasi untuk mengkonfigurasi autentikasi berbasis kunci di SSH. Tools ini umumnya telah terinstall di sebagian besar distribusi Linux. Namun, jika tidak ada di sistem Linux klien Anda, jalankan perintah berikut ini untuk menginstalnya:

CentOS/Fedora/RHEL

sudo yum -y install openssh

Ubuntu/Debian

sudo apt-get -y install openssh-client

2. Generate Pasangan Kunci

Jalankan perintah berikut untuk menghasilkan pasangan kunci publik/pribadi.

ssh-keygen

Anda akan diminta untuk memasukkan kata sandi. Anda dapat memasukkan ya/tidak tergantung kebutuhan Anda. Jika Anda memasukkan ya, maka Anda harus memasukkan frasa sandi untuk autentikasi.

$ ssh-keygen
    Generating public/private rsa key pair.
    Enter file in which to save the key (/home/centos/.ssh/id_rsa):
    Enter passphrase (empty for no passphrase):
    Enter same passphrase again:
    Your identification has been saved in /home/centos/.ssh/id_rsa.
    Your public key has been saved in /home/centos/.ssh/id_rsa.pub.
    The key fingerprint is:
    a6:d6:61:a6:83:0a:ba:8a:ed:bd:6a:d7:4f:cc:ae:75 centos@ip-172-31-23-73
    The key's randomart image is:
    +--[ RSA 2048]----+
    |                 |
    |                 |
    |                 |
    |                 |
    |        S        |
    |     . X .       |
    |.   ..= * E      |
    |oo.o...= .       |
    |*+=oo..oo        |
    +-----------------+

Disarankan agar memasukkan frasa sandi sehingga meskipun seseorang memegang kunci pribadi Anda, mereka tidak akan dapat masuk ke sistem Anda tanpa memasukkan frasa sandi.

Perintah di atas akan menghasilkan file private key/kunci pribadi id_rsa dan file public key/kunci publik id_rsa.pub lalu akan tersimpan di folder .ssh.

3. Salin Kunci Publik ke Server Jarak Jauh

Selanjutnya Anda perlu menyalin public key ~/.ssh/id_rsa.pub ke remote server anda di ~/.ssh/authorised_keys

Ada beberapa cara untuk menyalin kunci publik ke server jauh. Pertama kita akan menggunakan perintah ssh-copy-id yang diinstal bersama dengan openssh-client. Jalankan perintah berikut untuk menyalin kunci publik ke server jauh. Ganti root dengan nama pengguna Anda, dan server-ip-addr dengan alamat IP server jauh Anda.

ssh-copy-id root@server-IP-addr

Ini akan mengatakan bahwa host Anda tidak diautentikasi, dan menanyakan apakah Anda akan melanjutkan. Ketik ya dan tekan enter.

$ ssh-copy-id root@53.36.153.13
    The authenticity of host '53.36.153.13 (53.36.153.13)' can't be established.
    ECDSA key fingerprint is 20:09:1b:a8:65:18:78:ab:56:cd:21:2f:24:ed:b1:74.
    Are you sure you want to continue connecting (yes/no)? yes

Selanjutnya, Anda harus memasukkan kata sandi nama pengguna Anda untuk server jarak jauh. ssh-copy-id akan memindai kunci publik dan secara otomatis memasukkan kunci publik ke server jauh.

Anda akan melihat output yang mirip dengan berikut ini.

/usr/bin/ssh-copy-id: INFO: attempting to log in with the new key(s), to filter out any that are already installed
/usr/bin/ssh-copy-id: INFO: 1 key(s) remain to be installed -- if you are prompted now it is to install the new keys
root@53.36.153.13's password:
Number of key(s) added: 1

4. Konfigurasikan Otentikasi Berbasis Kunci di SSH

Masuk ke SSH dengan perintah berikut dari klien Anda. Ganti root dengan nama pengguna Anda, dan server-ip-addr dengan alamat IP server jauh Anda.

ssh root@server-IP-address

Jika Anda telah menambahkan frasa sandi ke file kunci publik/pribadi Anda, maka Anda akan dimintai untuk mengisi frasa sandi tersebut.

Kemudian SSH akan secara otomatis memasukkan Anda menggunakan kunci pribadi dan otentikasi berbasis kunci.

Namun, Anda masih dapat masuk ke SSH hanya dengan menggunakan kata sandi seperti sebelumnya. Untuk menonaktifkan otentikasi berbasis kata sandi dan hanya menerapkan otentikasi berbasis kunci, edit file konfigurasi SSH default

sudo vi /etc/ssh/sshd_config

Perhatikan kode dibawah

PasswordAuthentication yes

Ganti menjadi

PasswordAuthentication no

5. Restart Apache Server

Restart Apache server untuk menerapkan perubahan.

CentOS/Fedora/RHEL

systemctl restart sshd

Ubuntu/Debian

service ssh restart

Anda disarankan untuk mengubah izin folder .ssh agar tidak ada orang lain yang dapat mengaksesnya.

chmod -R 600 ~/.ssh

Disarankan juga untuk membuat cadangan kunci publik Anda

cp ~/.ssh/id_rsa ~/.ssh/login_key

Anda dapat login ke SSH dengan kunci baru berikut ini:

ssh -i ~/.ssh/login_key root@server-IP-address
