Cara Setting Login SSH Passwordless/Tanpa Kata Sandi

Ferry Ananda Febian

Nov 28, 2022 • 2 min read

Umumnya ketika login ke SSH membutuhkan kata sandi sebagai autentikasi, tetapi Anda disini dapat login ke SSH tanpa menggunakan password/kata sandi.

Artikel ini akan menjelaskan bagaimana cara login ke SSH tanpa password.

Cara Setting Login SSH Passwordless/Tanpa Kata Sandi

Berikut adalah langkah-langkah untuk setting login SSH tanpa kata sandi. Pada dasarnya, Anda perlu membuat kunci publik dan menambahkannya ke file ~/.ssh/authorized_keys host server.

1. Cari kunci SSH yang sudah ada

Buka terminal dan jalankan perintah berikut untuk memeriksa apakah ada kunci SSH di sistem Anda. Karena artikel ini tidak ingin menimpa kunci SSH yang sudah ada sebelumnya.

ls -al ~/.ssh/id_*.pub

Jika perintah di atas mencantumkan kunci apa saja, maka Anda dapat menggunakan kunci tersebut dan melewati langkah berikutnya. Jika Anda melihat pesan “No such file or directory” itu berarti tidak ada kunci SSH di sistem Anda.

2. Buat Key Pair Baru

Jalankan perintah berikut untuk membuat pasangan kunci SSH. Ganti email@domain.com dengan alamat email administrator situs web Anda.

sudo ssh-keygen -t rsa -b 4096 -C "email@alamat-domain.com"

Selanjutnya Anda akan melihat prompt pertanyaan, tekan enter untuk menggunakan file path dan lokasi default. Di beberapa kasus, username dapat diganti dengan username Linux.

Enter file in which to save the key (/home/username/.ssh/id_rsa):

Selanjutnya, Anda akan melihat prompt yang menanyakan apakah Anda ingin memasukkan kata sandi. Jika Anda tidak ingin menggunakan kata sandi maka tekan enter saja. Dalam hal ini, Anda tidak akan dimintai kata sandi setiap kali masuk melalui SSH. Jika Anda memerlukan login SSH otomatis, disarankan untuk menggunakan login SSH tanpa frasa sandi.

Enter passphrase (empty for no passphrase):

Anda akan melihat pesan bahwa kunci Anda telah dibuat dan disimpan di /home/username/.ssh/id_rsa. Di sini nama pengguna akan berbeda tergantung pada nama pengguna Linux Anda.

Jalankan perintah berikut untuk membuat daftar private key dan public key Anda.

ls ~/.ssh/id_*

Output:

/home/username/.ssh/id_rsa /home/username/.ssh/id_rsa.pub

3. Salin Public Key

Selanjutnya Anda perlu menyalin public key ke server. Anda dapat menggunakan perintah ssh-copy-id. Ganti remote_user dengan remote user Anda sendiri dan juga server_ip sesuai ip remote server Anda.

ssh-copy-id remote_user@server_ip

Setelah Anda mengautentikasi, perintah di atas akan secara otomatis menambahkan public key Anda ke file authorized_keys pengguna server.

Jika sistem Anda tidak memiliki perintah ssh-copy-id , gunakan perintah berikut untuk menyalin kunci publik.

sudo cat ~/.ssh/id_rsa.pub | ssh remote_username@server_ip_address "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

3. Login ke remote server

Jalankan perintah berikut untuk login ke remote server

sudo ssh remote_user@server_ip

4. Matikan Autentikasi Kata Sandi

Setelah Anda login ke server jauh menggunakan public key, buka file berikut. Lakukan langkah ini jika Anda berhasil masuk ke server jauh menggunakan public key. Jika Anda tidak dapat masuk menggunakan public key dan jika Anda menonaktifkan otentikasi kata sandi, maka Anda akan dikunci dari sistem Anda.

sudo nano /etc/ssh/sshd_config

Cari baris berikut dan ubah nilainya menjadi ‘no’.

PasswordAuthentication no 
ChallengeResponseAuthentication no 
UsePAM no

Simpan dan keluar dari file. 

5. Restart Layanan SSH

Restart layanan SSH untuk menerapkan perubahan.

Ubuntu/Debian

sudo systemctl restart ssh

Redhat/Fedora/CentOS

sudo systemctl restart sshd
