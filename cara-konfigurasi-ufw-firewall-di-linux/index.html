Cara Konfigurasi UFW Firewall di Linux

Ferry Ananda Febian

Nov 18, 2022 • 2 min read

UFW atau singkatan dari Uncomplicated Firewall adalah layanan firewall di sistem Linux yang mengizinkan buka/tutup port dan memonitor traffic di jaringan.

UFW ini cukup mudah digunakan untuk memanagement iptables rules sesuai kebutuhan Anda.

Artikel ini akan membahas bagaimana cara konfigurasi UFW Firewall di sistem Linux.

Cara Konfigurasi UFW Firewall di Linux

Secara default UFW sudah terinstall di Ubuntu, jika belum terinstall gunakan perintah berikut untuk install UFW pada sistem Anda.

sudo apt install ufw

Harap dicatat, secara default UFW akan memblokir semua koneksi yang masuk dan mengizinkan semua koneksi keluar, artinya tidak ada seorang pun yang dapat terhubung dengan server anda, sementara semua aplikasi dapat mengirim data keluar, oleh karena itu Firewall rule perlu diatur secara eksplisit.

UFW policies tersimpan di /etc/default/ufw dan dapat diubah dengan perintah ini:

sudo ufw default <policy> <chain>

Setelah UFW terinstall, aktifkan untuk menjalankan service

sudo ufw enable

Berikut adalah perintah umum menggunakan UFW

Get UFW status

Cek status UFW:

sudo ufw status

Disable UFW

Disable layanan UFW

sudo ufw disable

Reset UFW

Perintah ini digunakan untuk mereset rules UFW:

sudo ufw reset

Application Profiles

Setiap kali Anda menginstal paket menggunakan perintah apt, hal ini akan membuat profil aplikasi di ufw, yang terdiri dari deskripsi layanan dan rules firewall untuk aplikasi tersebut. Semua profil aplikasi disimpan di /etc/ufw/applications.d dan dapat di list menggunakan perintah berikut:

sudo ufw app list

Available applications:   

NGINX
uwsgi
python3-gunicorn

Jika Anda ingin melihat informasi spesifik tentang aplikasi misal NGINX, gunakan perintah ini:

sudo ufw app info 'Nginx'

Allow HTTP Port 80

Perintah ini digunakan untuk mengizinkan port 80 melewati firewall UFW

sudo ufw allow http

ATAU

sudo ufw allow 80/tcp

Ketika Anda menulis nomor port atau range port, Anda harus menetapkan spesifik protokol yang digunakan, misal TCP atau UDP

Allow SSH Port 22

Perintah ini akan mengizinkan SSH dengan port 22 melewati firewall UFW

sudo ufw allow ssh

ATAU

sudo ufw allow 22/tcp

Allow HTTPS Port 443

Perintah ini mengizinkan https port 443 melewati firewall UFW

sudo ufw allow https

ATAU

sudo ufw allow 443/tcp

Allow Custom Port 3939

Perintah ini mengizinkan port 3939 melewati firewall UFW

sudo ufw allow 3939/tcp

Allow Port Range 21-22

Perintah ini akan mengizinkan port dengan range 21-22 FTP server melewati firewall UFW, gunakan titik dua sebagai pemisah range port

sudo ufw allow 21:22/tcp

Allow from specific IP address

Perintah ini mengizinkan IP spesifik 34.34.34.12 melewati firewall UFW

sudo ufw allow from 34.34.34.12

Allow from specific IP on specific port

Perintah ini akan mengizinkan akses dari IP spesifik beserta nomor port nya

sudo ufw allow from 34.34.34.12 to any port 22

Allow subnets

Anda juga dapat mengizinkan subnets spesifik berdasarkan range IP

sudo ufw allow from 34.43.21.0/24

Deny Access

Adalah perintah untuk memblokir akses dari port atau IP yang spesifik

sudo ufw deny http # block http port 80
sudo ufw deny https # block https port 443
sudo ufw deny ssh # block ssh port 22
sudo ufw deny 3939/tcp # block custom port 3939
sudo ufw deny from 34.43.21.12 # block access from 34.43.21.12
sudo ufw deny from 34.43.21.0/24 # block access from 34.43.21.1-34.43.21.254

Delete Rules

Anda juga dapat menghapus rules yang sudah ditetapkan sebelumnya di UFW firewall.

Contoh menghapus rule nomor 4

sudo ufw delete 4 # deletes rule number 4

Anda dapat melihat list rules beserta nomor dengan perintah berikut:

sudo ufw status numbered

Anda juga dapat menghapus rules selain dengan menyebutkan nomor rules:

sudo ufw delete allow 80/tcp
