Rangkuman Materi CCNA-1: Basic Device Configuration

Ferry Ananda Febian

Dec 21, 2022 • 3 min read

Device Name

Device name atau nama yang diberikan kepada perangkat Cisco seperti router atau switch. Nama ini bertindak sebagai identifier unik untuk perangkat tersebut dan biasanya digunakan dalam konfigurasi jaringan atau saat mengakses perangkat melalui CLI (Command Line Interface).

Di dalam Basic Device Configuration Cisco, device name biasanya dikonfigurasi pada bagian hostname dengan perintah “hostname” seperti berikut:

hostname NAMA_PERANGKAT

Contoh:

hostname Router1

Dengan mengkonfigurasi device name, Anda dapat dengan mudah mengidentifikasi perangkat di dalam jaringan Anda dan memudahkan pengelolaan jaringan secara keseluruhan. Selain itu, device name juga bisa digunakan sebagai parameter dalam beberapa perintah CLI untuk mengakses atau mengelola perangkat tersebut.

Password Guidelines

Password guidelines pada basic device configuration Cisco adalah aturan yang diberikan oleh Cisco untuk memastikan keamanan password yang digunakan pada perangkat Cisco. Berikut ini beberapa aturan terkait password guidelines pada basic device configuration Cisco:

  1. Password harus terdiri dari minimal 8 karakter, terdiri dari huruf besar, huruf kecil, angka, dan simbol.
  2. Password harus terdiri dari kombinasi karakter yang tidak mudah diungkap, seperti kombinasi kata-kata yang terlalu umum atau mudah diingat.
  3. Password harus diubah setiap 90 hari, sehingga tidak mudah ditebak oleh pihak yang tidak bertanggung jawab.
  4. Password harus disimpan dengan aman, tidak boleh dibagikan kepada orang lain atau dituliskan di tempat yang mudah terlihat.
  5. Password harus dihapus setelah digunakan untuk keperluan tertentu, seperti akses ke perangkat Cisco.

Dengan mengikuti aturan-aturan tersebut, maka password yang digunakan pada perangkat Cisco akan lebih aman dan tidak mudah ditebak oleh pihak yang tidak bertanggung jawab.

Configure Passwords

Configure Passwords adalah proses penyetelan password pada perangkat Cisco. Password ini digunakan untuk mengakses perangkat tersebut dan melakukan konfigurasi pada perangkat tersebut.

Untuk melakukan konfigurasi password pada perangkat Cisco, pertama-tama kita harus masuk ke mode konfigurasi perangkat dengan mengetikkan “enable” pada prompt. Kemudian, masukkan perintah “configure terminal” untuk masuk ke mode konfigurasi terminal. Setelah itu, gunakan perintah “enable secret [password]” untuk mengkonfigurasi password yang akan digunakan untuk masuk ke mode privileged. Perintah ini akan mengenkripsi password yang telah ditentukan sehingga tidak terlihat oleh orang lain.

Selain itu, kita juga bisa mengkonfigurasi password yang akan digunakan untuk masuk ke mode user dengan menggunakan perintah “username [nama user] password [password]” di terminal. Setelah itu, jangan lupa untuk menyimpan perubahan konfigurasi dengan mengetikkan “write memory” pada terminal.

Dengan mengkonfigurasi password pada perangkat Cisco, kita dapat membatasi akses ke perangkat tersebut dan melindungi konfigurasi yang telah dibuat. Sebaiknya gunakan password yang kuat dan tidak mudah ditebak oleh orang lain agar keamanan perangkat terjaga dengan baik.

Sintaks dasar Configure Passwords:

Sw-Floor-1# configure terminal
Sw-Floor-1(config)# line console 0
Sw-Floor-1(config-line)# password cisco
Sw-Floor-1(config-line)# login
Sw-Floor-1(config-line)# end
Sw-Floor-1#

Berikut ini adalah contoh konfigurasi password pada perangkat Cisco:

  • Masuk ke mode konfigurasi perangkat dengan mengetikkan “enable” pada prompt.
  • Masuk ke mode konfigurasi terminal dengan mengetikkan “configure terminal”.
  • Konfigurasi password untuk masuk ke mode privileged dengan mengetikkan “enable secret [password]”. Misalnya, jika password yang ingin digunakan adalah “cisco123”, maka kita akan mengetikkan “enable secret cisco123”.
  • Konfigurasi password untuk masuk ke mode user dengan mengetikkan “username [nama user] password [password]”. Misalnya, jika nama user yang ingin digunakan adalah “admin” dan password yang ingin digunakan adalah “admin123”, maka kita akan mengetikkan “username admin password admin123”.
  • Simpan perubahan konfigurasi dengan mengetikkan “write memory”.

Encrypt Passwords

Encrypt Passwords adalah fitur yang memungkinkan password pada perangkat Cisco untuk dienkripsi atau diacak sehingga tidak terlihat oleh orang lain. Password yang telah dienkripsi tidak bisa dibaca dengan mudah oleh orang lain dan hanya bisa diakses dengan menggunakan kata sandi yang sesuai.

Fitur Encrypt Passwords biasanya digunakan untuk mengkonfigurasi password yang akan digunakan untuk masuk ke mode privileged pada perangkat Cisco. Untuk mengaktifkan fitur ini, kita bisa menggunakan perintah “enable secret [password]” pada terminal. Perintah ini akan mengenkripsi password yang telah ditentukan sehingga tidak terlihat oleh orang lain.

Selain itu, fitur Encrypt Passwords juga bisa digunakan untuk mengkonfigurasi password yang akan digunakan untuk masuk ke mode user. Perintah yang digunakan untuk mengaktifkan fitur ini adalah “username [nama user] secret [password]”. Setelah itu, jangan lupa untuk menyimpan perubahan konfigurasi dengan mengetikkan “write memory” pada terminal.

Sintaks dasar Encrypt Passwords:

Sw-Floor-1# configure terminal
Sw-Floor-1(config)# service password-encryption
Sw-Floor-1(config)#

Banner Messages

Banner Messages adalah pesan atau informasi yang ditampilkan ketika seseorang mengakses perangkat Cisco. Pesan ini ditampilkan sebelum user login ke perangkat tersebut dan bisa berupa pesan selamat datang, pesan warning, atau informasi lain yang diinginkan.

Pada Basic Device Configuration Cisco, kita bisa mengkonfigurasi banner messages dengan menggunakan perintah “banner [type] [message]”. [Type] merupakan tipe banner yang akan dibuat, sedangkan [message] adalah pesan yang akan ditampilkan. Ada beberapa tipe banner yang bisa digunakan, yaitu:

  • Login banner: ditampilkan sebelum user login ke perangkat.
  • Motd banner: ditampilkan setelah user login ke perangkat.
  • Exec banner: ditampilkan saat user memasuki mode exec.

Contoh konfigurasi banner messages pada perangkat Cisco adalah sebagai berikut:

Sw-Floor-1# configure terminal
Sw-Floor-1(config)# banner motd #Authorized Access Only#
