Cron Job Linux: Pengertian dan Cara Kerja

Ferry Ananda Febian

Jul 28, 2022 • 2 min read
Cron Job adalah sebuah perintah yang dapat dijalankan secara otomatis secara berkala. Misal kita ingin menjalankan sebuah fungsi di jam atau waktu tertentu, dengan pemanfaatan cron job, tugas administrator tentu jadi lebih efektif dan juga efisien.

Cron Job di Linux

Setiap sistem Linux memungkinkan user untuk mengatur kapan cron akan dieksekusi pada interval khusus user. Interval ini bisa menit, jam, hari, minggu, bulan, hari dalam satu minggu, atau kombinasi dari semua nilai interval. Cron job biasanya digunakan untuk administrasi dan pemeliharaan sistem linux, database atau situs web. Misalnya seperti rutin melakukan back up, memeriksa pembaruan, mengirim email, menginstal patch dan sebagainya. Perintah yang digunakan dalam pekerjaan cron dapat berupa skrip shell atau perintah di terminal linux.

Cara Kerja Cron Job

Semua perintah cron job dieksekusi dalam file teks Linux yang disebut crontab. User dapat membuat, melihat, bahkan mengedit file ini menggunakan perintah crontab -e .

Cara Ribet Menggunakan Cron Job

Cara ribet yang dimaksud ane adalah mempelajari segala sintaks yang tersedia pada cron job.

Sintaks Cron Job

Setiap perintah cron job perlu ditentukan terpisah dalam baris file crontab dan harus mengikuti sintaks berikut. Cron Job terdiri dari 6 bagian yang dipisahkan oleh spasi, 5 bagian pertama adalah bagian yang berbasis waktu dan bagian terakhir adalah perintah yang akan dijalankan.

untuk penjelasan lebih lanjut bisa simak sintaks dibawah ini

* * * * * command(s)
  ^ ^ ^ ^ ^
  | | | | | allowed values
  | | | | | -------
  | | | | ----- Day of week (0 - 7) (Sunday=0 or 7)
  | | | ------- Month (1 - 12)
  | | --------- Day of month (1 - 31)
  | ----------- Hour (0 - 23)
  ------------- Minute (0 - 59)

Lima operator penting Cron Job

Operator Cron Job digunakan untuk mendefinisikan waktu eksekusi job.

  • (*) tanda bintang berarti setiap kejadian. Jika ditempatkan di bagian pertama, cron berarti menandakan waktu dalam satuan menit. Jika ditempatkan di kolom kedua, cron berarti menandakan waktu dalam satuan jam, dan seterusnya.
  • (â€“) tanda hubung memungkinkan user untuk menentukan rentang nilai. Misalnya, 1-5 di bagian pertama berarti cron job akan berjalan setiap menit selama 5 menit pertama setiap jam. Agak ribet ya jelasinnya? semoga paham hehe, oke lanjut.
  • (,) koma memungkinkan user untuk menentukan daftar nilai. Misalnya, menentukan 15,30,45 di kolom pertama berarti cron akan berjalan pada menit 15, 30, 45 setiap jam.
  • (/) slash memungkinkan user untuk menentukan langkah interval. Misalnya */10 di kolom pertama berarti tugas akan berjalan setiap 10 menit.
  • User bisa menggunakan 3 operator terakhir secara bersamaan seperti 1-5,10,30-40/2 di kolom pertama artinya setiap jam, tugas akan berjalan setiap menit selama 5 menit pertama, lalu pada menit 10, dan kemudian setiap 2 menit mulai dari menit 30 sampai menit 40.

Contoh penggunaan cron job

0 0 1 * * /home/user/backup.sh
0 10 * * * service apache2 restart

Jalankan Cron Job Setiap 5 Menit

Berikut adalah perintah sederhana untuk menjalankan cron job setiap 5 menit.

*/5 * * * * command

Kolom pertama menjelaskan bahwa tugas cron akan dijalankan setiap 5 menit. Kolom berikutnya memiliki tanda bintang untuk menentukan tugas yang perlu dijalankan, baik setiap jam, hari, minggu, dan bulan.
