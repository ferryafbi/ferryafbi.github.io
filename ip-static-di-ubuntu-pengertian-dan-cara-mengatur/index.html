IP Static di Ubuntu: Pengertian dan Cara Konfigurasi

Ferry Ananda Febian

Oct 29, 2022 • 3 min read

Terkadang Anda mungkin perlu mengkonfigurasi alamat IP static di Linux. IP static merupakan lawan kata dari IP dinamis atau dhcp, sederhananya IP static tidak akan berubah-ubah layaknya IP dinamis. Pada artikel ini, kita akan belajar cara mengatur IP static di Ubuntu menggunakan tools Netplan. Anda dapat menggunakannya untuk mengubah alamat IP secara permanen dari terminal linux.

Cara Konfigurasi IP Static di Ubuntu

Berikut adalah langkah-langkah untuk mengatur IP static di Ubuntu menggunakan Netplan.

Netplan memungkinkan Anda untuk mengkonfigurasi antarmuka jaringan menggunakan file YAML sebagai gambaran antarmuka. Setelah Anda membuat file YAML (.yaml), Netplan akan membuat file konfigurasi sesuai alat perender pilihan Anda. Netplan mendukung dua layanan – NetworkManager dan Systemd-networkd. Selanjutnya kita akan mendemonstrasikan menggunakan networkd.

Anda dapat mengatur satu IP static, beberapa alamat IP statis, atau menyesuaikan rentang IP. Disini kami akan mengatur rentang IP static 192.168.100.0-192.168.100.255.

1. Identifikasi Antarmuka Ethernet

Jalankan perintah berikut untuk mengidentifikasi antarmuka ethernet yang ingin Anda konfigurasi.

ip link

Anda akan melihat output seperti berikut, yang mencantumkan semua antarmuka jaringan yang tersedia, satu antarmuka loopback dan yang lainnya adalah antarmuka ethernet.

1: lo:  mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
2: ens2:  mtu 1500 qdisc fq_codel state UP mode DEFAULT group default qlen 1000
    link/ether 08:00:27:6c:13:63 brd ff:ff:ff:ff:ff:ff

Sebagai contoh, kita akan menggunakan antarmuka ens2. lo adalah antarmuka loopback dan tidak dapat digunakan untuk tujuan ini.

2. Edit File Konfigurasi Netplan

Saat Anda menginstal Netplan, secara otomatis akan membuat beberapa file YAML (.yaml) di /etc/netplan.

Jika tidak ada file YAML di /etc/netplan di sistem Anda, jalankan perintah berikut untuk membuatnya.

sudo netplan generate

Anda dapat mengedit file yang tersedia di /etc/netplan. Bisa menggunakan nano atau Vim editor, disini kami mengedit file menggunakan vim.

sudo vim /etc/netplan/01-netcfg.yaml

Tambahkan baris konfigurasi berikut dalam huruf tebal di bawah bagian ethernet. Ganti ens2 dengan nama antarmuka ethernet Anda yang ingin diupdate..

network:
version: 2
  renderer: networkd
  ethernets:
    ens2:
      dhcp4: no
      dhcp6: no
      addresses:
        - 192.168.100.21/24
      gateway4: 192.168.100.1
      nameservers:
          addresses: [8.8.8.8, 1.1.1.1]

Anda perlu menentukan rentang IP dalam notasi CIDR. Misalnya, kami telah menetapkan 192.168.100.21/24 untuk rentang IP 192.168.100.0 – 192.168.100.255. Gunakan IP calculator untuk mempermudah.

Sekarang bisa kita lihat setiap baris yang ditambahkan.

dhcp4: no dan dhcp6: no, artinya DHCP dinonaktifkan

addresses: Tentukan rentang alamat IP static di bawah alamat: Bisa juga menambahkan satu atau lebih alamat IP IPv4 atau IPv6 untuk ditetapkan ke antarmuka jaringan. Anda juga dapat menyebutkan rentang IP.

gateway4: Tentukan gateway atau gerbang agar tersambung dengan koneksi yang berbeda

nameserver: Mengatur alamat IP dari nama server.

Simpan dan tutup file.

3. Terapkan perubahan

Jalankan perintah berikut untuk menerapkan perubahan.

sudo netplan apply

Verifikasi konfigurasi yang telah dilakukan dengan menjalankan perintah berikut. Ganti ens2 dengan antarmuka ethernet yang ingin Anda modifikasi.

ip addr show dev ens2

Setelah menekan enter, Anda akan melihat output berikut ini. Bagian yang dicetak tebal menunjukkan rentang IP statis yang dikonfigurasi untuk antarmuka jaringan Anda.

2: ens2: <broadcast,multicast,up,lower_up> mtu 1500 qdisc mq state UP group default qlen 1000
    link/ether 08:00:27:6c:13:63 brd ff:ff:ff:ff:ff:ff
    inet 192.168.121.221/24 brd 192.168.121.255 scope global dynamic ens3
       valid_lft 3575sec preferred_lft 3575sec
    inet6 fe80::5054:ff:feb0:f500/64 scope link 
       valid_lft forever preferred_lft forever
</broadcast,multicast,up,lower_up>

Oke, jadi seperti itu cara konfigurasi IP static pada ubuntu, mudah kan? hehe
