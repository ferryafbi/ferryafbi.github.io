Cara Install NFS Server dan Client di CentOS

Ferry Ananda Febian

Nov 18, 2022 • 2 min read

NFS atau singkatan dari Network File System adalah distribusi file sistem yang mengizinkan penggunanya untuk sharing file dan direktori ke pengguna lain di jaringan yang sama. NFS juga dapat diakses secara remote via SSH untuk mengakses file dan direktori seolah-olah file tersebut tersedia secara lokal.

Artikel ini akan membahas bagaimana cara install NFS Server dan Client di CentOS.

Langkah-Langkah Install NFS Server

Perlu dicatat, IP dibawah ini hanya sebagai contoh, Anda dapat menyesuaikannya sesuai kebutuhan.

NFS Server IP: 54.34.13.14 
NFS Clients IPs: start dari 54.16.3.0/24 dan seterusnya

1. Install NFS Server

Buka terminal dan jalankan perintah install NFS Server

dnf install nfs-utils

2. Enable dan Start NFS Server

Jalankan perintah berikut untuk enable dan start NFS Server

systemctl start nfs-server.service 
systemctl enable nfs-server.service 
systemctl status nfs-server.service

Perintah terakhir untuk melihat status NFS Server

3. Buat File Sistem

Selanjutnya adalah membuat file sistem test kemudian export dan share ke NFS server.

mkdir -p  /mnt/nfs_shares/test 
mkdir  -p /mnt/backups 
ls -l /mnt/nfs_shares/

4. Export direktori

Untuk export direktori, perlu menambahkan konfigurasi pada path /etc/export

sudo vi /etc/exports

Tambahkan baris ini untuk mengizinkan akses khusus clients pada range IP 54.16.3.0/24

/mnt/nfs_shares/test   54.16.3.0/24(rw,sync) 
/mnt/backups 54.16.3.0/24(rw,sync,no_all_squash,root_squash) 

Berikut adalah penjelasan konfigurasi diatas:

  • rw – allows read & write to these folders
  • sync – allows NFS to write operations when requested
  • no_all_squash – map GIDs and UIDs from client to identical ones on NFS server
  • root_squash – map requests from root user to UID/GID 0

Selanjutnya jalankan perintah export direktori

exportfs -arv

-a , digunakan untuk mengekspor semua

-r , ekspor ulang semua direktori

-v, output verbose

5. Update Firewall

Update firewall rules untuk mengizinkan NFS server menerima request dari remote clients:

firewall-cmd --permanent --add-service=nfs 
firewall-cmd --permanent --add-service=rpc-bind 
firewall-cmd --permanent --add-service=mountd
firewall-cmd --reload

6. Setup NFS Clients

Buka terminal pada sistem clients dan jalankan perintah instalasi NFS client

dnf install nfs-utils nfs4-acl-tools         [On CentOS/RHEL] 
sudo apt install nfs-common nfs4-acl-tools   [On Debian/Ubuntu]

7. Daftar exported file sistem

Jalankan perintah berikut untuk menampilkan daftar folder yang di export NFS server.

showmount -e 54.34.13.14

8. Buat Lokal Direktori

Buat direktori lokal pada NFS client untuk dipasang ke direktori remote dari server NFS. File yang dipasang berformat ntf

mkdir -p /mnt/backups 
mount -t nfs  54.34.13.14:/mnt/backups /mnt/backups

Konfirmasi dengan menjalankan perintah mount dan grep nfs

mount | grep nfs

Anda akan melihat output folder sudah ter mount

Selanjutnya jalankan perintah untuk menambahkan entry di /etc/fstab  dan buat mount persistent bekerja setiap sistem selesai reboot.

echo "54.34.13.14:/mnt/backups     /mnt/backups  nfs     defaults 0 0">>/etc/fstab

Untuk unmount file ini, jalankan perintah berikut dari NFS client.

umount /mnt/backups

9. Test semua konfigurasi

Jalankan perintah berikut ini untuk melihat apakah file di server dapat diakses oleh client:

touch /mnt/backups/file_created_on_server.text     [On NFS Server] 
ls -l /mnt/backups/file_created_on_server.text     [On NFS client]

Sekarang coba buat file pada client dan cek apakah bisa dilihat pada server:

touch /mnt/backups/file_created_on_client.text     [On NFS Client] 
ls -l /mnt/backups/file_created_on_client.text     [On NFS Server]

Selesai, Anda sudah membuat NFS Server dan client beserta instalasi dan konfigurasi pada sistem.
