Cara Menghapus Images, Containers dan Volumes di Docker

Ferry Ananda Febian

Nov 23, 2022 • 4 min read

Docker dapat membuat aplikasi dan layanan terisolasi sehingga memudahkan pengguna untuk menjalankannya dimana saja. Saat menggunakan Docker, banyak nya image yang di unduh, containers dan Volumes data yang tidak terpakai tentu hanya akan memenuhi ruang penyimpanan.

Artikel ini akan membahas cara menghapus images, containers dan volume yang tidak digunakan di Docker.

Cara Menghapus Images, Containers dan Volumes di Docker

Docker menyediakan satu perintah yang akan membersihkan sumber daya apapun seperti images, containers, volume dan jaringan yang tidak terpakai (tidak diberi tag atau tidak dikaitkan dengan containers):

docker system prune

Untuk menghapus lebih lanjut containers yang dihentikan dan semua gambar yang tidak digunakan (bukan hanya image yang tidak digunakan), tambahkan opsi -a ke perintah:

docker system prune -a

Menghapus Docker Images

Hapus satu atau lebih images yang spesifik

Gunakan perintah docker images dengan tambahan opsi -a untuk menemukan ID images yang ingin Anda hapus. Opsi ini akan menampilkan semua daftar images yang diunduh. Saat Anda menemukan images yang ingin dihapus, Anda dapat meneruskan ID atau tagnya ke docker rmi:

List:

docker images -a

Hapus:

docker rmi Image Image

Hapus Images Dangling/Menggantung

Images Docker terdiri dari beberapa lapisan. Images yang menggantung/dangling adalah lapisan yang tidak memiliki hubungan dengan images yang diberi tag. 

Images tersebut tidak digunakan dan hanya akan memenuhi ruang penyimpanan. Images menggantung dapat ditemukan dengan opsi filter -f dengan nilai dangling=true setelah perintah docker images, setelah itu gunakan perintah docker image prune untuk menghapus.

Catatan: Jika Anda membuat images tanpa tag, image tersebut akan muncul di daftar images menggantung karena tidak ada hubungannya dengan images yang diberi tag. Sangat direkomendasikan menambahkan tag saat membuat images, selain itu Anda tetap dapat menambahkan tag di images Docker dengan perintah docker tag.

List:

docker images -f dangling=true

Remove:

docker image prune

Hapus Images Sesuai Pola

Anda dapat menemukan semua images yang cocok dengan pola menggunakan kombinasi docker images dan grep. Setelah itu Anda dapat menghapusnya dengan menggunakan awk lalu ID images akan diteruskan ke docker rmi. Perhatikan bahwa perintah ini tidak disediakan oleh Docker dan belum tentu tersedia di semua sistem:

List:

docker images -a |  grep "pattern"

Hapus:

docker images -a | grep "pattern" | awk '{print $3}' | xargs docker rmi

Hapus Semua Images

Semua image Docker pada sistem dapat dicantumkan dengan menambahkan opsi -a ke perintah docker images. Setelah itu, Anda dapat menambahkan opsi -q untuk meneruskan ID gambar ke docker rmi:

List:

docker images -a

Hapus:

docker rmi $(docker images -a -q)

Menghapus Containers

Hapus satu atau lebih Containers yang spesifik

Gunakan perintah docker ps dengan menambahkan opsi -a untuk menampilkan semua daftar containers, setelah itu gunakan perintah rm dengan menyebutkan ID atau nama containers

List:

docker ps -a

Hapus:

docker rm ID_or_Name ID_or_Name

Hapus Containers Saat Keluar

Anda dapat membuat dan menjalankan containers hanya satu kali, artinya ketika selesai digunakan, containers tersebut akan terhapus secara otomatis ketika keluar:

Jalankan dan hapus:

docker run --rm image_name

Hapus Semua Containers yang Keluar

Anda dapat melihat daftar containers dengan perintah docker ps -a dan memfilternya berdasarkan status: created, restarting, running, paused, atau exited. Untuk meninjau daftar containers yang exit/keluar, tambahkan opsi -f untuk memfilter berdasarkan status. Gunakan opsi -q untuk memverifikasi containers yang dihapus lalu ID akan diteruskan ke perintah docker rm:

List:

docker ps -a -f status=exited

Hapus:

docker rm $(docker ps -a -f status=exited -q)

Hapus Containers Menggunakan Lebih Dari Satu Filter

Filter Docker dapat digabungkan dengan menambahkan lebih dari satu filter. Cara ini akan memproses daftar containers yang memenuhi kedua kondisi tersebut. Misalnya, Anda ingin menghapus semua containers yang ditandai dengan created (keadaan ini dapat terjadi saat Anda menjalankan Containers dengan perintah yang tidak valid) atau exited, Anda dapat menggunakan dua filter:

List:

docker ps -a -f status=exited -f status=created

Hapus:

docker rm $(docker ps -a -f status=exited -f status=created -q)

Hapus Containers Sesuai Pola

Anda dapat menampilkan semua containers yang cocok dengan pola menggunakan kombinasi docker ps dan grep. Setelah itu Anda dapat menggunakan awk dan xargs untuk meneruskan ID ke docker rm. Perhatikan bahwa tools ini tidak disediakan oleh Docker dan belum tentu tersedia di semua sistem:

List:

docker ps -a |  grep "patternâ€

Hapus:

docker ps -a | grep "pattern" | awk '{print $1}' | xargs docker rm

Stop dan Hapus Semua Containers

Anda dapat melihat daftar container yang berjalan dengan perintah docker ps. Tambahkan opsi -a untuk menampilkan semua daftar containers. Setelah itu Anda dapat menambahkan opsi -q untuk meneruskan ID ke perintah docker stop dan docker rm:

List:

docker ps -a

Hapus:

docker stop $(docker ps -a -q)
docker rm $(docker ps -a -q)

Hapus Volumes

Hapus satu atau lebih volume tertentu di Docker versi 1.9 dan terbaru

Gunakan perintah docker volume ls untuk menemukan nama volume yang ingin Anda hapus. Kemudian hapus dengan perintah docker volume rm :

List:

docker volume ls

Hapus:

docker volume rm volume_name volume_name

Hapus volume Dangling/Menggantung Docker versi 1.9 dan terbaru.

Karena volume harus berdiri sendiri dari containers, ketika containers dipindahkan, volume tidak secara otomatis dipindahkan secara bersamaan. Ketika ada volume dan tidak lagi terhubung ke container manapun, itu disebut volume yang menggantung/dangling. Untuk menemukannya serta mengkonfirmasi untuk dihapus, Anda dapat menggunakan perintah docker volume ls dengan filter untuk membatasi hasil pada volume yang menggantung/dangling. Setelah itu Anda dapat menghapus semuanya dengan docker volume prune:

List:

docker volume ls -f dangling=true

Hapus:

docker volume prune

Keluarkan Containers dan Volumenya

Jika Anda membuat volume tanpa nama, maka dapat dihapus bersamaan dengan containers dan menambahkan opsi -v. Perlu diingat bahwa ini hanya berfungsi dengan volume yang tidak diberikan nama. Ketika containers berhasil dihapus, ID-nya akan ditampilkan. Perhatikan bahwa tidak ada referensi yang dibuat untuk menghilangkan volume. Jika tidak disebutkan namanya, maka proses ini akan dihapus secara tersembunyi dari sistem. Jika diberi nama, maka volume tersebut tetap ada secara tersembunyi.

Hapus:

docker rm -v container_name

Kesimpulan

Artikel ini membahas beberapa perintah umum yang digunakan untuk menghapus image, containers, dan volume di Docker. Ada banyak kombinasi dan opsi lain yang dapat digunakan di setiap kasus. Untuk panduan lebih lengkapnya Anda mungkin dapat melihat dokumentasi resminya di website Docker untuk docker system prune, docker rmi, docker rm, dan docker volume rm di sini docker (base command)
