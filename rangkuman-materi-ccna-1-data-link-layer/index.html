Rangkuman Materi CCNA-1: Data Link Layer

Ferry Ananda Febian

Jan 4, 2023 • 2 min read

Pengertian Data Link Layer

Data Link Layer merupakan lapisan kedua dari model OSI (Open Systems Interconnection). Tujuan dari lapisan ini adalah untuk menyediakan layanan jaringan yang dapat diandalkan antar perangkat jaringan yang terhubung secara langsung.

Fungsi-fungsi utama dari lapisan ini adalah:

  • Memastikan bahwa data yang dikirimkan dari satu perangkat ke perangkat lainnya dapat diterima dengan benar dengan menggunakan teknik-teknik error checking seperti checksum dan parity bit.
  • Menangani masalah-masalah yang terjadi pada jalur fisik, seperti collision, noise, dan interference.
  • Menyediakan mekanisme enkapsulasi data agar dapat diterima oleh perangkat jaringan yang lain.
  • Menyediakan layanan jaringan yang dapat diandalkan, seperti flow control dan error recovery.

Lapisan Data Link sangat penting untuk menjamin kinerja jaringan yang stabil dan tepat waktu. Tanpa lapisan ini, data tidak dapat di transmisi dengan benar dari satu perangkat ke perangkat lainnya, sehingga menyebabkan kerusakan data dan kegagalan komunikasi.

Perbandingan Topologi

Berikut ini adalah perbandingan karakteristik dari metode media access control pada topologi WAN (Wide Area Network) dan LAN (Local Area Network):

  1. Broadcast domains: Pada topologi WAN, broadcast domain biasanya lebih luas dibandingkan pada topologi LAN, karena WAN terdiri dari beberapa LAN yang terhubung melalui internet atau jaringan telepon. Hal ini menyebabkan penggunaan metode media access control yang lebih efisien pada topologi WAN, seperti metode token passing atau metode polling.
  2. Bandwidth: Pada topologi WAN, bandwidth biasanya lebih rendah dibandingkan pada topologi LAN, karena WAN menggunakan jalur transmisi yang terbatas, seperti internet atau jaringan telepon. Hal ini menyebabkan penggunaan metode media access control yang lebih terbatas pada topologi WAN, seperti metode Carrier Sense Multiple Access (CSMA) atau metode CSMA/CD.
  3. Jumlah perangkat: Pada topologi LAN, jumlah perangkat biasanya lebih banyak dibandingkan pada topologi WAN, karena LAN terbatas pada area yang kecil. Hal ini menyebabkan penggunaan metode media access control yang lebih kompleks pada topologi LAN, seperti metode CSMA/CD atau metode Token Ring.
  4. Latency: Pada topologi WAN, latency biasanya lebih tinggi dibandingkan pada topologi LAN, karena WAN menggunakan jalur transmisi yang lebih jauh dan terbatas. Hal ini menyebabkan penggunaan metode media access control yang lebih efisien pada topologi WAN, seperti metode token passing atau metode polling.
  5. Keamanan: Pada topologi LAN, keamanan biasanya lebih tinggi dibandingkan pada topologi WAN, karena LAN terbatas pada area yang kecil dan lebih mudah dijaga. Hal ini menyebabkan penggunaan metode media access control yang lebih kompleks pada topologi LAN, seperti metode token ring atau metode CSMA/CD.

Data Link Frame

Data link frame adalah paket data yang dikirimkan melalui lapisan Data Link dari model OSI (Open Systems Interconnection). 

Berikut ini adalah karakteristik dan fungsi dari data link frame:

  1. Karakteristik: Data link frame terdiri dari header, payload, dan trailer. Header berisi informasi yang diperlukan untuk mengirim dan menerima data link frame, seperti alamat MAC (Media Access Control) pengirim dan penerima, tipe protokol, dan lain-lain. Payload berisi data yang dikirimkan, sedangkan trailer berisi informasi yang diperlukan untuk mengecek integritas data, seperti checksum atau parity bit.
  1. Fungsi: Data link frame memiliki beberapa fungsi utama, diantaranya adalah:
  • Mengirim dan menerima data melalui jalur fisik: Data link frame digunakan untuk mentransmisikan data dari satu perangkat ke perangkat lainnya melalui jalur fisik, seperti kabel koaksial atau fiber optik.
  • Menangani masalah-masalah jalur fisik: Data link frame menangani masalah-masalah yang terjadi pada jalur fisik, seperti collision, noise, dan interference.
  • Menyediakan layanan jaringan yang dapat diandalkan: Data link frame menyediakan layanan jaringan yang dapat diandalkan, seperti flow control dan error recovery, sehingga data dapat dikirim dan diterima dengan benar.
  • Mengenkapsulasi data: Data link frame mengenkapsulasi data agar dapat diterima oleh perangkat jaringan yang lain dengan benar.
