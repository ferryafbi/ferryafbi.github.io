Cara Melihat Log dari Crontab di Linux

Ferry Ananda Febian

Jan 6, 2023 • 2 min read

Crontab adalah sebuah program yang menjadwalkan tugas-tugas yang ditentukan untuk dijalankan pada waktu tertentu. Di Linux, crontab dapat digunakan untuk menjalankan script atau perintah secara otomatis sesuai dengan jadwal yang ditentukan. Crontab juga dapat digunakan untuk menjalankan task yang diulang setiap beberapa menit, jam, atau hari. Namun, tahukah Anda bagaimana Cara Melihat Log dari Crontab di Linux? Berikut pembahasannya.

Untuk melihat log dari crontab di Linux, Anda bisa menggunakan perintah grep CRON /var/log/syslog. 

grep CRON /var/log/syslog

Perintah ini akan mencari semua entri yang terkait dengan crontab di file syslog, yang biasanya berisi log dari berbagai sistem dan aplikasi di sistem Anda.

Jika Anda ingin melihat log crontab untuk user tertentu, Anda bisa menambahkan opsi -u ke perintah grep seperti ini: grep CRON -u nama_user /var/log/syslog. 

grep CRON -u nama_user /var/log/syslog

Perintah ini hanya akan menampilkan log crontab yang dijalankan oleh user tersebut.

Anda juga bisa menggunakan perintah cat untuk membaca file syslog secara langsung. Misalnya: cat /var/log/syslog. 

cat /var/log/syslog

Namun, output dari perintah ini mungkin sangat panjang dan sulit dibaca, terutama jika Anda hanya ingin melihat log crontab.

Selain itu, Anda juga bisa menggunakan perintah crontab -l untuk melihat daftar crontab yang sedang dijalankan oleh user yang bersangkutan. 

crontab -l

Perintah ini akan menampilkan daftar task yang sedang dijadwalkan, termasuk waktu dan interval yang telah ditentukan.

Demikian artikel tentang Cara Melihat Log dari Crontab di Linux, semoga membantu.
